# 执行规则

会按“**任务强度 → 对应思考强度**”来执行，不再一律用同一档。

按这套方式落地：

- **L1 轻量任务**
  - 用于：整理、抄录、汇总、格式化、简单核对
  - 做法：快速直出，少做扩展推演
  - 例如：汇总 weekly snapshot、整理指标表、更新文档目录

- **L2 标准任务**
  - 用于：常规对比、归类、交叉核对、轻度解释
  - 做法：做必要检查和上下文比对，但不过度展开
  - 例如：月初/月末指标对比、治理动作归类、已完成项收口

- **L3 深度任务**
  - 用于：多源证据综合、异常闭环判断、风险分析、方案设计
  - 做法：会做更系统的交叉验证，防止误判
  - 例如：判断 orphan / declared-only 是否成立、判断某项是否应降级保护、设计月度收口结构

- **L4 审慎任务**
  - 用于：高风险决策、方向裁决、是否升级专项治理
  - 做法：采用保守判断，明确边界、前提和风险，不草率下结论
  - 例如：是否重启专项治理、是否调整治理策略、是否把高风险项直接落地

后续我会在两层都体现这个规则：

1. **执行层**
   - 做每个任务时，按任务复杂度切换对应思考强度

2. **输出层**
   - 在文档、报告、提示词里，把每个任务对应的思考强度标出来，便于你和其他 agent 对齐

简单说就是：

- 能轻做的不重做
- 该深入的会深入
- 该保守的地方会保守

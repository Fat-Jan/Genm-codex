# Genm-codex `v1.3` Roadmap

> Status: `archived-mainline-roadmap`
>
> 这份 roadmap 仍保留在根目录，作为 `v1.4` 的直接上游参考；相关延后方向说明已归档到 `docs/90-归档/阶段/v1.3-deferred-directions.md`。

## 目的

这份 roadmap 承接 `v1.2` 收口后的下一阶段工作。

它回答：

- `v1.2` 之后最值得继续推进的增强是什么
- 哪些属于默认工作流与质量链主线
- 哪些虽然能做，但不该进入 `v1.3` 主线

## 状态说明

- `[done]`：已完成
- `[in_progress]`：进行中
- `[planned]`：已确认要做
- `[blocked]`：受前置条件阻塞
- `[deferred]`：明确延后

## 联网标签

- `[local]`：本地即可完成，不依赖外部资料
- `[hybrid]`：结构部分本地可做，但内容值或样本判断需要外部校准
- `[online]`：核心判断明显依赖外部资料或平台现状

## 当前总判断

`v1.3` 不应继续做“大而全补线”。

`v1.2` 已经完成了：

- `content-positioning` 第一轮落地
- `platform_positioning` 首批/第二梯队扩面
- 首批 profile 内容字段校准
- 主线 profile 的 legacy embedded reference 清理
- 第一轮 smoke 与交叉样本验证
- `novel-log` 去留判断

因此 `v1.3` 更应该聚焦：

1. **默认工作流稳定性与恢复体验**
2. **上游结构合同定义**
3. **质量路由一致性**
4. **高频技能规则栈去复制**
5. **`content-positioning` 第二轮扩面**
6. **Fanqie-first 样本冻结**

## 当前仍不建议拉进主线的方向

详细参考：

- [v1.3-deferred-directions.md](/Users/arm/Desktop/vscode/Genm-codex/docs/90-归档/阶段/v1.3-deferred-directions.md)

- 不重开 monolithic runtime / plugin system
- 不把 MCP memory 升成项目真值层
- 不把 `novel-scan` 偷渡进默认工作流
- 不让 `novel-log` 重新回到默认主线
- 不单独做“纯治理、脱离用户价值”的 registry/manifest 工程

原因：

- `monolithic runtime / plugin system`
  - 当前默认骨架仍是 `docs + skills + scripts + sidecars`
  - 现在重开统一 runtime 的成本高、迁移面大、对写作者直接价值低
- `MCP memory` 升格为项目真值层
  - 会制造第二真值中心
  - 与本地文件真值原则冲突，恢复和审计成本也会明显上升
- `novel-scan` 偷渡默认主线
  - 它的价值在外部 research 候选
  - 但证据质量、时效性、可重复性仍不稳定，适合继续做实验链，不适合变成默认推荐链
- `novel-log` 回到默认主线
  - 当前已经作为专家辅助能力有明确定位
  - 除非默认工作流开始强依赖 trace 日志，否则没有必要重新开一条主线
- 脱离用户价值的纯治理工程
  - registry / manifest 只有在明确服务 consumer contract 时才值得做
  - 不能为了“更像架构”而单独起线

## 推荐顺序

1. 主线 B：默认工作流稳定性与恢复体验
2. 主线 F 的前置定义块：`F1-F3`

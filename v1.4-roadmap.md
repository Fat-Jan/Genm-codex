`v1.4` 不是从零开始。

- `start-here / skill-usage / default-workflows` 入口补线

5. 把回归样本自动化

## 当前总判断

`v1.4` 最值得推进的，不是“继续堆新能力”，而是把已经存在但仍半联动的能力做稳。

当前最有价值的 5 条主线是：

1. **核心主线稳定化**
2. **规则压缩与文档去百科化**
3. **结构预防前移**
4. **流派 / bucket 真实打通**
5. **自动化回归样本**

## 完成后怎么标记状态

每项任务完成后，至少补 4 个信息：

1. `status`
   - 改成 `[done]` / `[blocked]` / `[in_progress]`
2. `files`
   - 改了哪些文件
3. `verify`
   - 跑了什么验证命令
4. `commit`
   - 对应 commit hash（如果已进主仓或功能分支）

建议格式：

```markdown
- Status: `[done]`
- Owner: `[me]`
- Review: `[review:mimo]`
- Verify:
  - `pytest ...`
  - `bash scripts/validate-migration.sh`
- Commit:
  - `abcd123`
```

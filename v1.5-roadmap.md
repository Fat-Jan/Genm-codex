# Genm-codex `v1.5` Roadmap

> Status: `active-mainline-roadmap`
>
> 这是当前主线 roadmap，继续保留在根目录作为高可见入口。

## 目的

`v1.5` 不再延续 `v1.4` 的“主线基础设施补洞”节奏，而是进入：

1. 消费层产品化
2. 样本库与回归治理
3. profile / bucket / tagpack 的 registry 化
4. `novel-scan` 的 contract 化与 consumer 接线

重点不是“再堆能力”，而是：

- 把已有能力变成稳定消费面
- 把扩面工作交给 `mimo` 时，先把 contract / guardrail / verify 门定死
- 明确哪些东西在 `v1.5` 只做准备，不做最终实现

---

`v1.4` 主线已完成，当前主仓基线：

- `workflow-health / quality-audit / knowledge-projection`

- `e2e-novel` 与《嫁妆单》真实 smoke / regression 样本

## 总判断

`v1.5` 里最该优先做的，不是直接大扩面，而是先完成一组 **codex-first 前置任务**，把后续能交给 mimo 的范围做实。

也就是说：

- `profile` 可以扩
- `novel-scan` 可以推进
- `skill merge` 可以做准备
- `runtime/plugin` 可以做边界研究

但这些都不该在没有 contract / verify / non-goal 的情况下直接开做。

当前这份 roadmap 对“方向判断”已经够用，但如果要真正开始分工执行，还必须明确三件事：

1. `codex-first` 任务各自要产出什么文件
2. 每个前置任务跑什么验证才算完成
3. 完成后，`mimo` 是被解锁了哪一组任务

下面的 `P0` 已按这个标准补齐。

- `profile`：可以扩，但先定 contract
- `novel-scan`：可以推进，但先定 contract，不直接主链化

- `runtime/plugin`：先做边界研究，不做实现

# Genm-codex `v1.0.0-rc1` 退出条件

> 历史阶段文档：本文记录的是 `v1.0.0` 正式发布前的 RC 退出条件；当前正式状态以 [v1-final-decision.md](/Users/arm/Desktop/vscode/Genm-codex/docs/00-当前有效/v1-final-decision.md) 为准。

## 目的

定义：

- 什么算 RC 通过
- 什么算 RC 挂起
- 什么算必须退回 `0.x`

---

## 通过条件

满足以下全部条件，才能从 RC 进入正式 `v1.0.0`：

1. 默认工作流跑通
2. 包装层判断稳定
3. 质量路由稳定
4. 样本项目闭环稳定
5. 文档入口一致
6. 没有新增 blocker

---

## 挂起条件

满足以下任一项，RC 应挂起，不进入正式版：

1. 默认工作流能跑，但关键判断不稳定
2. 包装与正文承载关系仍明显飘
3. `review / precheck` 结论在相似样本间差异过大
4. RC 期间发现默认主线仍依赖实验能力

---

## 退回条件

满足以下任一项，RC 应直接退回 `0.x`：

1. 默认工作流主线出现断裂
2. 样本项目闭环无法复现
3. 文档入口重新出现明显冲突
4. 关键 skill 行为与文档承诺严重不符

---

## 正式 `v1.0.0` 前最后确认

正式发布前，至少还要确认：

1. 默认稳定能力清单最终版
2. 实验能力清单最终版
3. README / start-here / skill-usage 的最终一致性
4. 最终 release note 中没有把实验能力写成默认能力

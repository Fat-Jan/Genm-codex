# 开篇方法与剧情层次规则层设计

## 背景

当前仓库已经把“方法论文档 -> skill 接线 -> 轻量 state 写回”跑通在反脸谱化和质量门上，但还缺一组更基础、更通用的“开篇方法 + 剧情层次”规则层。

目标不是补一套课程笔记，而是补一组默认工作流可消费的规则：

- `novel-outline` 在起盘和章纲阶段能锁开篇承诺与剧情线层次
- `novel-write` 在正文阶段能避免“只会往前推、不知道在推什么”
- `novel-review` / `novel-precheck` 能把开篇抓力、层次清晰度、推进有效性纳入检查
- `novel-package` 能把正文实际承载力和开篇包装承诺接起来

## 设计原则

1. 承载层放在 `docs/`，作为项目内单一事实源。
2. 第一版做跨题材通用骨架，不预设某单一 bucket 为默认真理。
3. 题材特化只先定义接口，不在第一版里铺满各题材细则。
4. 不为该框架新造平行 state 中心，只允许在现有 review 元数据里做轻量增强。
5. 规则按需读取，不要求每个 skill 每次全量吞完整套文档。

新建：

## 模块职责

### 1. 开篇目标与成功标准

定义开篇不是“交代背景”，而是完成：

- 主承诺可见
- 冲突前置
- 情绪入口成立
- 未解问题成立
- 前三章兑现路径可见

### 2. 开篇构件与组合公式

沉淀开篇常用构件：

- 冲突
- 悬念
- 情绪代入
- 信息差
- 主承诺
- 章末留钩

并给出可执行的组合顺序，而不是抽象术语堆砌。

### 3. 开篇故障与修正

统一识别这类失败模式：

- 背景先行、故事后到
- 只有设定没有事件
- 只有压迫没有交换
- 承诺过大但近期无兑现
- 章末没有下一步驱动

### 4. 剧情层次模型

定义项目级的层次拆法：

- 主线
- 支线
- 人物线
- 关系线
- 世界/规则线

并要求 outline 和 chapter outline 至少知道“这一章主要推进哪一层，触动了哪一层”。

### 5. 推进链与残账设计

把“事件推进”拆成可检验链条：

- 事件发生
- 信息差变化
- 条件交换
- 代价落地
- 残账留存

### 6. 题材特化接口

不给具体题材规则全文，只定义以后每个题材应该补哪 4 类信息：

- 开篇钩子偏好
- 推进密度偏好
- 高频故障
- 预检重点

## Skill 接线策略

### `novel-outline`

读取重点：

- `README.md`
- `01-开篇目标与成功标准.md`
- `04-剧情层次模型.md`
- `05-推进链与残账设计.md`

在总纲和章纲阶段显式锁：

- 开篇承诺
- 黄金三章的第一轮兑现
- 当前章主推进线
- 次级被触动线
- 本章残账

### `novel-write`

在前三章或开篇强化场景时，优先读：

- `01`
- `02`
- `03`

在多线或过渡章时，优先读：

- `04`
- `05`

### `novel-review`

将该框架作为结构检查层之一，重点看：

- 开篇抓力
- 层次清晰度
- 推进是否只有“发生了事”但没有账本变化

允许把以下键作为 `dimension_scores` 的可选增强项：

- `开篇抓力`
- `层次清晰度`
- `推进有效性`

### `novel-precheck`

作为投稿前风险检查：

- 开头首屏是否真的给到主承诺/冲突
- 前三章是否有硬兑现
- 剧情推进是否过顺
- 关系/信息/代价是否留残账

### `novel-package`

把包装承诺和正文承载力接起来，避免：

- 简介承诺极强但前三章正文接不住
- 开篇包装只会喊爆点，不会说明兑现节点

## State 策略

第一版不新增顶层 state 区块。

只做两件事：

1. 在 `shared/references/shared/state-schema.md` 里补可选评分键说明
2. 允许 `novel-review` 在 `chapter_meta[N].dimension_scores` 中按需写入：
   - `开篇抓力`
   - `层次清晰度`
   - `推进有效性`

## 验证策略

### 红灯验证

先加一个测试，要求：

- 新框架目录与文件存在
- 5 个 skill 已引用新框架
- 入口文档已暴露新事实
- state schema 已记录可选评分键

### 绿灯验证

实现后运行：

- `python -m unittest tests.test_opening_plot_framework`
- `bash scripts/validate-migration.sh`

## 不做什么

- 第一版不做大规模题材资料库
- 第一版不把课程全文塞进 memory
- 第一版不新增独立 skill
- 第一版不要求 `shared/` 复制一套平行规则

## 区间分析

### 节奏

- 第1-3章完成了非常清楚的黄金三章：
  - 第1章：婚书被换，当场立局
  - 第2章：东宫问婚，原婚书回手
  - 第3章：太子开口，规则抬升
- 第4-6章没有掉回“重复撕扯”，而是持续换战场：
  - 庄子旧人线
  - 外院账房线
  - 伯爷书房线
- 这说明这条线不只是开头能炸，后续也能继续抬层级

### 桶适配

- 主桶 `宫斗宅斗` 的核心驱动力是：
  - 压制关系
  - 身份反咬
  - 阵营与规矩
  - 章末挂更大局
- 这 6 章基本都在服务这套逻辑
- `恶女` 叠加后的效果也成立：
  - 不是纯狠
  - 不是发疯
  - 而是“守规矩表皮下的稳、狠、会算”

### 连续性

- 婚书 -> 东宫 -> 庄子旧人 -> 外院账房 -> 伯爷书房
  这条升级链路是通的
- 主角手里的证据链也在持续累积：
  - 原婚书
  - 玉扣
  - 旧账册
  - 东宫令签
  - 顾家旧回帖
- 当前第6章章末把悬念推进到“半页实账仍在府中流转”，足够支持后续继续写

## 最强优势

- 开头快，压制关系明确，番茄女频长篇的第一屏吸附力是够的。
- `恶女` 叠加后没有把主桶写歪，反而强化了“后宅守规矩式反咬”的爽点。
- 三章之后仍能持续升级，不是只会重复“正厅对骂”。
- 主线已经成功从柳氏单线抬到伯爷层，说明扩写空间成立。

## 主要风险

- 目前的强项是“局势升级”，但感情侧 / 关系侧还没开始承载更丰富的长期追读点，后续需要注意不要只剩查账推进。
- 当前更高优先级风险不是节奏，而是上游设定门缺失：
  - `庶女 + 庶妹 + 嫡庶压制 + 二姑娘` 同时出现，但谱系未闭合
  - 关系词被直接推入标题和包装，导致外层承诺先于设定成立
  - 主角名与标题缺少投稿前最小检索门

## 预检判断

### 综合评估

- 作为**方向候选**，它仍值得继续做。
- 作为**当前内部生产模板**，它不再够格。
- 作为**当前就拿去外投的首批样本**，现在不够格。

### 必须修复

- 补一层最小家族谱系真值：
  - 谁是正室 / 继室 / 妾室
  - 谁是嫡出 / 庶出
  - `二姑娘` 之前的人物去向
- 把书名、简介、总纲、角色卡、正文称谓统一到同一套真值
- 对主角名和外层标题加最小撞名 / 高相似度检索门

### 建议修复

- 完成谱系修正后，再跑一轮集中 `review / precheck`
- 给后续第7章开始的推进，预留一条更强的人物关系压力线，不要只剩账线推进

### 包装状态

- `packaging-needs-update: yes`
- 当前书名、婚书局、东宫开口、恶女反咬这些承诺并非都不可用
- 但“庶妹 / 嫡庶压制 / 二姑娘”这一组外层词已经进入危险区，必须先回到设定层重做

### 投稿建议

- `do-not-submit`

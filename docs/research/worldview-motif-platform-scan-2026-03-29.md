# 世界观母题平台扫描（番茄 / 阅文-起点 / 晋江）

> 日期：2026-03-29
> 目的：判断“世界观母题层”是否值得在 Genm-codex 中落地，以及首批母题应该优先吸收哪些平台信号。

## 结论

- **值得做一个新的“世界观母题层”**。
- 这个新层的第一形态更适合是：
  - `shared/references/` 下的母题目录
  - `creative-brief` 的新增输入段
  - `novel-init / novel-outline / novel-package` 的上游参考
- **不建议第一步就做新 skill**。
  - 当前缺的是“立项输入与灵感来源的标准化层”，不是又一个和 `novel-setting` 抢职责的设定编辑器。

一句话判断：

- 平台已经稳定证明“题材桶 + 脑洞机制 + 关系包装”是有效的开书输入。
- 但平台给的多是活动赛道、分类口径和热词，不可直接当项目真值。
- 最合理的做法，是把这些平台信号压缩成一个 **母题卡目录**，再接入上游结构。

## 平台共识：真正稳定的不是“单个设定”，而是“母题簇”

把三平台信号折叠后，比较稳定的不是单个点子，而是以下几簇：

### 1. 规则 / 副本 / 生存机制母题

平台证据：

- 番茄：`悬疑脑洞` 明确吸收 `恐怖无限流 / 恐怖游戏逃生`
- 阅文：`生命只剩七天` 这种短篇创意 premise
- 晋江：`未来游戏悬疑`、`快穿`、幻想向关系文长期稳定存在

可沉淀成的母题：

- 时间循环
- 生命倒计时
- 规则怪谈
- 无限流副本
- 恐怖游戏逃生
- 失忆轮回

### 2. 都市机制外挂母题

平台证据：

- 番茄：`都市脑洞` 明确点名 `签到 / 直播 / 神豪 / 文娱 / 奶爸`
- 阅文：`系统流` 被纳入奇幻脑洞征文
- 晋江：幻想现言与现代关系线经常包裹系统、穿书、快穿

可沉淀成的母题：

- 系统绑定
- 签到成长
- 神豪身份跃迁
- 直播驱动剧情
- 文娱逆袭
- 职业规则反转

### 3. 历史 / 古代机制脑洞母题

平台证据：

- 番茄：`历史脑洞` 直接支持 `科举 / 争霸 / 系统 / 空间`
- 晋江：`古代穿越 / 古言脑洞 / 奇幻言情` 分类稳定存在
- 阅文现实与历史线长期支持“题材真实度 + 机制创新”并存

可沉淀成的母题：

- 科举外挂
- 古代工业化
- 争霸升级
- 女扮男装仕途
- 古代读心 / 系统 / 空间

### 4. 科幻 / 末世 / 星际社会实验母题

## 首批母题目录，建议分三层

如果要把“世界观母题层”做成可消费的 repo 资产，建议不要只做一个平铺清单，而是拆成三层：

### A. 世界观母题

定义：

- 直接改变世界底层运行方式的 premise

示例：

- 时间循环
- 神话复苏
- 虚拟现实
- AI 统治
- 末日乌托邦
- 万物有灵
- 梦境入侵
- 高维囚笼

### B. 机制母题

定义：

- 不一定改变整个世界，但会强力驱动叙事推进的规则系统

示例：

- 倒计时
- 副本通关
- 规则试探
- 系统绑定
- 快穿任务
- 身份错位
- 记忆缺失
- 无限知识有代价

### C. 包装 / 关系母题

定义：

- 不是世界规则本身，而是决定作品点击口径与人物入口的叠加层

示例：

- 双男主 / 双女主
- 校园甜宠
- 娱乐圈综艺
- 科举仕途
- 宫斗宅斗
- 年代创业
- 行业商战
- 职业成长

## 不建议第一批就重押的母题

这些不是不能收，而是当前平台一手信号不够强，或者更适合作为 `v2 exploratory`：

- 暗物质文明
- 光与影二元文明战争
- 群体意识社会
- 宇宙牢笼
- 元素四元统治世界
- 无尽战争型纯宏大叙事

原因：

- 它们更像“作者私有高概念设定”
- 暂时不像平台明示的高频起盘口径
- 适合先放在母题卡库的 `探索类` 标签里，而不是 `v1 必读`

## 对 Genm-codex 的直接启发

### 1. 这个新层应该接在 `creative-brief` 前半段

原因：

- [上游结构合同](../00-当前有效/upstream-structure-contract.md) 已经把 `creative-brief` 定义成最早的立项输入层
- 平台信号也证明“开书灵感 / premise / 分类”属于写前输入，而不是写后补设定

建议新增字段：

- `世界观母题`
- `机制母题`
- `关系包装`
- `核心异常`
- `规则与代价`
- `隐藏真相`
- `首屏 world hook`

### 2. 首批不需要新 skill

更好的第一步：

- 新增：
  - `shared/references/writing/worldview-motif-catalog.md`
- 再改：
  - `shared/templates/project/creative-brief.md`
- 再让已有 skill 消费：
  - `novel-init`
  - `novel-outline`
  - `novel-package`

### 3. 这个母题层不该直接写入项目真值

母题层只负责：

- 给方向
- 给可选组合
- 给常见故障提醒

真正进入项目 `设定集/世界观/` 的，必须是已经被该项目选中的版本。

## 推荐顺序

1. 先基于本研究稿整理 `v1 母题卡目录`
2. 再给 `creative-brief` 增加母题字段
3. 再让 `novel-init / novel-outline / novel-package` 条件读取
4. 最后再判断是否需要一个单独的 `novel-premise` 或 `novel-concept` skill

## 当前建议

当前阶段最稳的路线是：

- **先做“母题目录 + brief 接口”**
- **暂不做新 skill**

这能直接服务：

- 开书灵感
- 组合题材立项
- 包装前置
- 世界观丰富时的灵感补给

同时不会把仓库主线从“稳定上游结构”拉偏到“继续堆新 skill”。

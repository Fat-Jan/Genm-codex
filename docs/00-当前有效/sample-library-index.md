## 样本使用指南

### 1. Smoke 样本

- **用途**: 端到端测试、框架验证、回归测试
- **特点**: 结构完整、可重复执行、有明确的验证标准
- **使用方式**: 直接运行对应的 smoke 脚本

### 2. 真实项目样本

- **用途**: 真实场景验证、bucket 规则校准、质量参考
- **特点**: 真实项目、有实际创作内容、可作为质量基准
- **使用方式**: 参考其结构、规则应用方式、质量标准

### 3. 样本选择原则

- **验证框架**: 优先使用 smoke 样本
- **验证 bucket**: 优先使用对应 bucket 的真实项目样本
- **验证质量**: 优先使用高置信度样本（如庶女谋略）

## 样本维护

### 1. 新增样本

后续维护默认遵循：

1. 先更新 `shared/templates/sample-manifest-v1.json`
2. 再更新本索引页
3. 如需要，再更新矩阵页和治理文档

### 2. 更新样本

1. 先更新 manifest 中的状态、分层或 regression 目标
2. 再更新本索引页的人读说明
3. 如是 regression 样本，确认其退化目标仍清晰

### 3. 清理样本

1. 先确认样本不再被 smoke / regression / docs / tests 引用
2. 先更新 manifest，再更新本索引页
3. 如有必要，再补归档说明

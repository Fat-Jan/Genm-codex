## 一、复检结论

**思考强度：L4 审慎**

本轮结论：**通过，样本库 / regression 治理已经具备长期维护面，但仍建议以后以“manifest-first + regression-target-first”继续收口，而不是回到索引页驱动。**

当前状态可概括为：

1. machine-readable 真值入口已经存在，并由 `shared/templates/sample-manifest-v1.json` 承担
2. 人读说明层已经分为：
   - 样本治理文档
   - 样本索引页

3. regression 样本和 smoke 样本都已经有真实测试入口，不再只是文档 inventory
4. bucket sample 覆盖面已形成首批多题材落点

   - 后续新增样本是否继续 manifest-first

因此，这一方向当前不需要重开结构实施，更适合进入**维护规则固定 + 持续复检**阶段。

## 三、当前缺口与风险

**思考强度：L4 审慎**

当前没有发现需要立即改代码的结构缺口，但有 3 个长期维护风险需要固化口径：

### 3.1 新增样本可能再次只更新索引，不更新 manifest
风险：
- 人读索引与 machine-readable 真值重新漂移

### 3.2 新 bucket 样本可能存在“覆盖了题材，但没有明确 verification target”
风险：
- 样本只是示例，不足以进入 regression / smoke 体系

### 3.3 regression 样本可能继续增加，但“防什么退化”不够明确
风险：
- 测试能跑，但样本为什么存在会越来越模糊

这些风险当前都还是**维护面风险**，不是结构失效。

---

## 四、建议固定的长期维护规则

**思考强度：L3 + L4**

后续建议固定以下规则：

1. **manifest-first**
   - 新增样本先补 `sample-manifest-v1.json`
   - 再补索引页和矩阵页

2. **regression-target-first**
   - 如果样本被归为 `regression`，必须写清楚它要防什么退化
   - 最好在 manifest 中保留 `verification_targets`

3. **bucket coverage 不等于 regression coverage**
   - bucket sample 的首要职责是“覆盖题材落点”
   - 不能默认它就承担 regression 样本职责

4. **索引页继续降级为浏览层**
   - `sample-library-index.md` 可以保留
   - 但不再承担真值入口职责

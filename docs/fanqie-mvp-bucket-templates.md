## 1. 现实情感

### 定位

- 轨道：`shortform` 优先，也可扩到中短
- 目标：高情绪、高共鸣、高转化

### 读者驱动

- 痛点代入
- 不公压迫
- 反击 / 觉醒

### 推荐标签包

- `出轨反击`
- `婚恋压迫`
- `家庭控制`
- `原生家庭创伤`
- `女性觉醒`
- `离婚反杀`

### 标题模板

- 人物关系 + 核心伤害 + 反击结果
- 社会痛点 + 高情绪词 + 逆转

示例结构：

- `丈夫出轨后，我把婆家送上热搜`
- `女儿抢我房子那天，我决定不再当妈`

### 简介模板

1. 第一句：先抛最痛的伤害
2. 第二句：再抛主角被逼到什么程度
3. 第三句：亮出反击动作或觉醒转折
4. 第四句：留下更大的关系反噬

### 开头规则

- 前 `100-200` 字必须出现伤害现场
- 不先铺背景
- 先给读者情绪立场

### 爽点 / 付费点周期

- 开头即痛点
- 很快给第一次反击
- 中段连续加码
- 结尾用更大的关系反噬或社会性后果收束

### precheck 重点

- 情绪是不是够集中
- 痛点是不是够快
- 反击是不是够狠
- 结尾是不是够“想看后果”

## 2. 宫斗宅斗

### 定位

- 轨道：`longform`
- 目标：强关系链、强算计、强连续追读

### 读者驱动

- 压迫与翻盘
- 阵营斗法
- 身份抬升

### 推荐标签包

- `嫡庶压制`
- `继母陷害`
- `假千金/真千金`
- `赐婚/退婚`
- `后宅翻盘`
- `权谋联姻`

### 标题模板

- 身份反差 + 压制关系 + 高反转
- 明确阵营 + 明确被害 + 明确翻盘点

示例结构：

- `被继母活埋后，我成了全府最不能惹的人`
- `庶妹抢我婚事那夜，太子先跪了`

### 简介模板

1. 被谁压
2. 压到什么程度
3. 主角靠什么翻盘
4. 翻盘会撬动哪条更大的权力链

### 开头规则

- 第一屏必须见到压制关系
- 主角不能只是回忆受苦，要立刻在局里
- 第一章末就要给出第一次局内反击或破局准备

### 爽点 / 付费点周期

- 小翻盘
- 被更狠压制
- 身份提升
- 阵营站队
- 章末挂更大局

### precheck 重点

- 人物关系清不清
- 压制链是否够狠
- 主角翻盘是不是具体
- 章末有没有更大局势承接

## 3. 传统玄幻

### 定位

- 轨道：`longform`
- 目标：稳收、长线、可持续成长

### 读者驱动

- 底层逆袭
- 升级成长
- 世界展开

### 推荐标签包

- `废柴逆袭`
- `宗门压迫`
- `退婚羞辱`
- `父辈旧案`
- `金手指显性`
- `资源争夺`

### 标题模板

- 身份低位 + 机制金手指 + 生存压力
- 宗门/修行体系 + 逆袭承诺

示例结构：

- `废脉杂役，靠一面古镜杀穿宗门`
- `被逐前三日，我在后山捡到长生机缘`

### 简介模板

1. 主角低位身份
2. 倒计时 / 生存压力
3. 金手指规则收益
4. 更大的成长主线或旧案主线

### 开头规则

- 开头要快
- 第一章就见压迫
- 第一章就见金手指或翻盘机制
- 不要先做大段世界观

### 爽点 / 付费点周期

- 先受压
- 小反打
- 第一次升级 / 机缘
- 更大试炼 / 大比 / 秘境

### precheck 重点

- 金手指规则是不是显性
- 升级主线是不是清楚
- 身份反差是否足够强
- 章末承接是否能拉住读者

## 4. 现言甜宠

### 定位

- 轨道：`shortform` / `mid-short`
- 目标：高点击、高轻松感、高甜爽即时反馈

### 读者驱动

- 高糖关系
- 误会反差
- 轻松打脸

### 推荐标签包

- `先婚后爱`
- `协议恋爱`
- `误认身份`
- `豪门反差`
- `带崽闪婚`
- `白切黑男主`

### 标题模板

- 人物关系 + 意外事件 + 强甜反差

示例结构：

- `闪婚当天，我把京圈太子爷认成了司机`
- `离婚后，前夫天天装穷来追我`

### 简介模板

1. 快速给关系
2. 快速给误会
3. 快速给甜宠或打脸
4. 留下持续升级的情感钩子

### 开头规则

- 开场就进关系局
- 先抛误会或意外
- 不做慢铺感情线

### 爽点 / 付费点周期

- 开头即高糖或误会
- 很快见第一次反差
- 中段甜虐交替
- 结尾挂情感升级或身份翻转

### precheck 重点

- 甜点够不够快
- 误会够不够清楚
- 标题有没有关系张力
- 是否太像普通现言而不够可点

## 当前系统接入建议

第一批不要求一次把全部技能都接满。

最小可用顺序：

1. `package`
2. `precheck`
3. `outline`
4. `write`
5. `review`

也就是说，先让这几个桶至少拥有：

- 包装模板
- 预检口径
- 大纲口径
- 正文节奏规则

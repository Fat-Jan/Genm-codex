# 番茄内容桶约束清单

## 目的

这份文档回答：

- 一旦你确定某个番茄内容桶，
- 系统里到底该抽取哪些约束给：
  - `novel-outline`
  - `novel-package`
  - `novel-precheck`
  - 后续可能的正文链

它不直接写正文，而是定义**约束字段**。

---

## 一、跨桶共用约束

这些约束基本适用于番茄短故事和番茄高点击内容：

### 1. 开头速度

- 开头 `100` 字内要有明确看点
- 尽快直入主题
- 不要用长背景铺垫

### 2. 节奏密度

- 节奏必须紧
- 每一小段最好都在推进矛盾、悬念、反转或情绪
- 尽量不让读者有“停顿空间”

### 3. 标题

- 标题尽量接近 `30` 字上限，但不能臃肿
- 标题要：
  - 有人物关系
  - 有亮点 / 爆点
  - 有关键词
  - 有共情点

### 4. 转化节点

- 短故事尤其要重视试读节点
- 官方口径明确强调：
  - 试读比例
  - 解锁欲望
  - 渠道收益率

### 5. 真实收益指标

当前官方公开强调的短故事核心数据包括：

- 展现量
- 阅读量
- 点击率
- 集团渠道热度
- 集团渠道收益率
- 加书架量

这意味着：

- 你的包装和开头不是只服务“审美”
- 还服务：
  - 点击
  - 解锁
  - 转化

## 二、短故事共用约束

### 篇幅

- 合法区间：
  - `6000-80000`
- 最优区间：
  - `10000-30000`

### 结构

- 一篇完结
- 不靠长线连载留人
- 更靠：
  - 强开头
  - 强反转
  - 强情绪
  - 强结尾

### 视角

官方说明里，短故事常见：

- 第一人称
- 第三人称

但尤其强调：

- **第一人称代入感强**

### 排版

- 行文简短
- 适合手机阅读
- 不连篇累牍

### 人物

- 主要人物不宜过多
- 人设要极致
- 正反角色都要足够鲜明

### 冲突

- 多重压制
- 极强矛盾
- 一波三折

## 三、短故事桶级约束

### 1. 现言甜宠

#### 适配特征

- 小白
- 轻松
- 接地气
- 热门梗多

#### 约束建议

- 标题：
  - 人物关系 + 意外冲突
- 简介：
  - 强情感误会 + 快速甜虐点
- 正文：
  - 代入快
  - 口语化
  - 节奏紧
- precheck：
  - 看共情点是否足够快

### 2. 现实情感

#### 适配特征

- 真实
- 婚恋 / 婆媳 / 出轨 / 家庭伦理
- 情绪拉扯强

#### 约束建议

- 标题：
  - 社会痛点 + 人物关系冲突
- 简介：
  - 先抛痛点
  - 再抛反击或觉醒
- 正文：
  - 情绪推进快
  - 仇恨值拉得高
- precheck：
  - 看情绪是否够集中，是否够共鸣

### 3. 悬疑脑洞

#### 适配特征

- 第一人称强代入
- 离奇事件开篇
- 真相层层剥开

#### 约束建议

- 标题：
  - 离奇事件 + 强悬念
- 简介：
  - 不要讲清真相
  - 重点讲不合理事件
- 正文：
  - 开头就要有异常事件
  - 每段都要有悬念推进
- precheck：
  - 看悬念是否持续加压，而不是只靠气氛

### 4. 现实生活

#### 适配特征

- 新闻感
- 社会议题
- 强讨论度

#### 约束建议

- 标题：
  - 事件感强
  - 新闻感强
- 简介：
  - 先抛矛盾
  - 再抛现实议题
- 正文：
  - 冲突具体
  - 议题要落地
- precheck：
  - 看事件是不是够“能讨论”

## 四、长篇桶级约束（第一批建议）

### 1. 传统玄幻

#### 约束重点

- 身份反差
- 升级主线清晰
- 金手指规则显性
- 爽点兑现要快

### 2. 玄幻脑洞

#### 约束重点

- 高概念设定
- 机制感强
- 第一屏就要看到差异化点

### 3. 东方仙侠

#### 约束重点

- 修行逻辑
- 世界规则
- 机缘与代价
- 更强的修行成长感

## 五、建议在系统里怎么表示

后续真正接系统时，我建议至少拆成这些字段：

- `bucket_id`
- `bucket_group`
  - longform / shortstory
- `length_range`
- `preferred_pov`
- `opening_rule`
- `title_rule`
- `synopsis_rule`
- `payoff_cycle`
- `ending_rule`
- `distribution_note`
- `primary_metrics`

这样：

- `novel-package` 可直接吃标题/简介/包装规则
- `novel-precheck` 可直接吃桶级判断
- `novel-outline` 可吃结构规则

---

## 六、当前推荐执行顺序

1. 先挑 3 个最值得做的桶
2. 给每个桶写约束
3. 先接 `package`
4. 再接 `precheck`
5. 最后再看是否接正文链

我建议的优先顺序：

- 长篇：
  - `传统玄幻`
  - `玄幻脑洞`
- 短故事：
  - `现言甜宠`
  - `现实情感`
  - `悬疑脑洞`

# 工具包与工作流

## 目标

把反脸谱化体系变成可执行工作流，而不是只供阅读的理念文档。

## 推荐工作流

1. `outline`
   - 做角色分层
   - 建动力卡
   - 锁关系图和阵营分歧
   - 链接冲突与后果
2. `drafting`
   - 按场景单推进
   - 记录谁主动、谁误判、谁付账
3. `review`
   - 按检查单扫描
   - 输出路由建议
4. `repair`
   - 走 `fix` / `polish` / `rewrite`

## 轻量模式

适合短篇或支线：

- 1 张角色动力卡
- 1 张关系表
- 1 张场景单

## 全量模式

适合长篇、群像和多阵营线：

- 角色分层表
- 角色动力卡组
- 关系网络图
- 阵营分歧表
- 冲突/信息差/后果链表
- 场景推进单

## 模板

### 角色动力卡

```md

层级：核心 / 重要配角 / 功能
外显目标：
内隐需求：
核心恐惧：
资源：
约束：
误判：
底线：
变化触发器：
```

### 关系表

```md
A 与 B：
- 表层关系：
- 深层结构：
- 当前债务/依赖：
- 误判点：
- 未来可能分裂点：
```

### 阵营分歧表

```md
阵营：
- 共同目标：
- 路线 A：
- 路线 B：
- 成本容忍度差异：
- 对主角态度差异：
```

### 场景单

```md
场景目标：
主动方：
防守方：
误判点：
隐藏需求：
结尾变化：
残余成本：
```

### 快速修复卡

```md
症状：
判断依据：
最小修法：
是否需要 rewrite：
```

## 对 `novel-outline`

建议至少内化：

- 角色分层表
- 角色动力卡
- 关系图
- 冲突链

## 对 `novel-review`

建议至少输出：

- 关键症状
- 判断依据
- 推荐路由
- 最小修法

# 场景级群像推进

## 目的

把反脸谱化从“卷纲理念”变成“单场景执行动作”。

## 场景单

每个关键场景至少回答：

- 谁在主动想拿东西？
- 谁在守住什么？
- 谁误判了局势？
- 谁隐藏了真实需求？
- 场景结束后，什么状态发生了变化？

## 最小变化要求

关键场景结束后，至少改变一项：

- 信息
- 关系
- 立场
- 风险
- 资源
- 目标优先级

## 推荐披露顺序

1. 先给行为
2. 再给偏差
3. 再给动机
4. 最后给伤口或底线

这比一上来发人物说明书更容易让角色像“被逐步认识”。

## 场景级失衡信号

- 除主角外，其他人都只做反应
- 场景里每个人的语言风格和策略判断几乎一样
- 场景结束后，除了主角更强了，别的都没变

## 纠偏动作

- 给至少一个配角一次抢先行动
- 给至少一个角色一段不服务主角情绪的发言或行为
- 给一次胜利补上额外成本或后续麻烦

## 对 `novel-outline`

章纲里至少标注：

- 场景目标
- 主动方 / 防守方
- 误判点
- 场景余波

## 对 `novel-review`

按场景检查：

- 是否有人物只剩功能台词
- 是否有人物本可推进却被主角吃掉了行动权
- 是否存在“场景内推进，场景外归零”

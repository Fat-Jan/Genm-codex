# 角色动力系统

## 目的

把角色从“标签集合”改成“行动逻辑”。

## 五个基础字段

1. **欲望**
   - 这个角色眼下最想拿到什么？
2. **约束**
   - 为什么他不能直接拿到？
3. **误判**
   - 他看错了什么、低估了什么、害怕承认什么？
4. **代价**
   - 他每推进一步，会失去什么？
5. **变化**
   - 他会改变什么，或者坚持不改变什么？

## 角色动力卡

| 字段 | 说明 |
|------|------|
| 外显目标 | 当前阶段最明确的追求 |
| 内隐需求 | 他其实真正缺的东西 |
| 核心恐惧 | 他最怕失去或被证明的事 |
| 资源 | 他手里有什么 |
| 约束 | 谁/什么限制了他 |
| 误判 | 他错把什么当成真相 |
| 底线 | 他不愿跨过的线 |
| 变化触发器 | 什么会逼他改 |

## 常见错误

- 只有标签，没有目标
- 只有过去，没有现在
- 只有痛点，没有策略
- 只写“成长”，不写“代价”

## 设计建议

- 外显目标和内隐需求不要完全重合
- 误判最好能推着人物做错事，而不是只写成设定说明
- 变化线不一定是变好，也可以是固执到底、越陷越深、被迫转向

## 对 `novel-outline`

至少为核心角色和重要配角建立动力卡，避免只靠“人设标签”分工剧情。

## 对 `novel-review`

重点检查：

- 本章人物行为是否能被动力卡解释
- 是否存在“作者需要他这样做，所以他就这样做”
- 角色的误判有没有真正影响局势

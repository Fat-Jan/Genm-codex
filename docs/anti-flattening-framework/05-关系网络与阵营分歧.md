# 关系网络与阵营分歧

## 目的

让关系成为结构，而不是“喜欢/讨厌/忠诚/背叛”的平面标签。

## 关系的最低结构

关键关系至少要有两层：

- 表层关系：公开身份或公开态度
- 深层结构：债务、依赖、羞耻、误判、利用、保护欲、权力差

## 高价值关系类型

- 债务关系
- 依赖关系
- 利用关系
- 权力不对称
- 情感错位
- 同盟中的路线分歧
- “理解你，但仍反对你”

## 阵营分歧表

同一阵营至少要分出这些轴：

| 轴 | 说明 |
|----|------|
| 目标一致度 | 他们是否真的想要同一个结果 |
| 手段分歧 | 他们如何达到目标 |
| 代价容忍度 | 愿意牺牲到什么程度 |
| 红线 | 哪些做法不可接受 |
| 对主角态度 | 利用、合作、防备、试探、反对 |

## 失衡信号

- 整个阵营只会重复同一种立场
- 所有配角都以“是否支持主角”来定义
- 关系只剩“告白/翻脸/表忠心”

## 纠偏动作

- 给盟友增加不同的成本承受线
- 给反派阵营增加内部路线冲突
- 给亲密关系增加非爱情层面的负债或结构压力

## 对 `novel-outline`

要锁定：

- 核心关系图
- 至少一个联盟内分歧点
- 至少一个关系误判会在后续触发事件

## 对 `novel-review`

要检查：

- 同阵营角色是否说话逻辑过于相似
- 关系是否只在主角需要情绪时才存在
- 有没有关键关系在本章里发生了状态变化

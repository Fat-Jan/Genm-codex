# 反脸谱化体系

这组文档用于把“人物活人感、群像独立性、立场不脸谱、关系不按钮化”沉淀成可复用的项目规则层。

它不是某个单独题材的附加美德，而是跨流派的底层卫生标准。

## 使用原则

1. 先看 `01-总纲.md`，明确边界和核心原则。
2. 做大纲时，优先读上游结构包。
3. 做评审时，优先读下游检查包。
4. 做局部修复时，优先读修复包。
5. 规则按需读取，不要求每次全量吞 12 份。

如果你想直接看“这套东西在默认工作流里该怎么用”，优先读：

## 规则优先级

推荐按这个顺序应用：

1. canon / 用户要求 / `.mighty/state.json`
2. active bucket / 平台硬约束 / 已冻结 outline law
3. 反脸谱化体系
4. 技巧层优化与风味增强

这意味着反脸谱化体系负责“把人写回人”，但不能越权覆盖 canon、平台硬约束或已冻结真值。

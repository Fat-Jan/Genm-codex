# 冲突-信息差-后果链

## 目的

把“人物不脸谱”落到可推动剧情的结构里。

## 冲突来源

优先使用具体条件制造冲突：

- 资源不足
- 时间窗口
- 身份暴露风险
- 目标不完全一致
- 信息不对称
- 代价承担不均

## 信息差设计

至少区分三层：

1. **真相**
2. **各角色以为的真相**
3. **主角以为他知道的真相**

只要这三层不重合，人物就更不容易变成口号播放器。

## 后果链

一次行动后，至少改动一项：

- 关系
- 名声
- 资源
- 风险
- 阵营位置
- 信息状态

## “无作为”也是行为

角色不说、不救、不站队、不澄清，都应留下结果。

## 事件模板

一个最小有效事件通常包含：

1. 谁先想拿东西
2. 谁阻止或拖延
3. 谁误判局势
4. 谁因此付账
5. 下一步问题被怎样放大

## 失衡信号

- 冲突只靠吵架，没有条件差
- 证据、同盟、身份像系统自动掉落
- 本章推进很快，但没有留下残账

## 对 `novel-outline`

要锁定：

- 关键冲突的条件
- 关键信息差的归属
- 每一阶段行动的残余成本

## 对 `novel-review`

要检查：

- 冲突是否真的可失败
- 主角的成功是否太自动
- 本章是否存在“推进了，但没有任何结构变化”

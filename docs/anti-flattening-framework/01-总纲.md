# 反脸谱化总纲

## 目标

把“人物不只是功能件、阵营不只是喇叭、关系不只是按钮、主角不自动合法”变成稳定可执行的项目规则。

## 定义

这里的“脸谱化”不是指人物简单，而是指人物被压缩成单一用途：

- 只为主角提供信息
- 只为主角制造爽点
- 只为作者输出立场
- 只在被调用时存在

反脸谱化的核心，是让关键人物和关键阵营拥有独立逻辑、独立代价和独立余波。

## 不是这些东西

- 不是让所有角色都写得一样复杂
- 不是平均分戏
- 不是取消主角中心
- 不是禁止爽点
- 不是强迫所有冲突都灰度化

## 五条核心原则

1. **人先于功能**
   - 关键人物先回答“他是谁、要什么、怕什么”，再回答“他为主线做什么”。
2. **主角中心可以存在，但不能垄断世界**
   - 主角可以是故事轴心，但不能独占全部解释权和后果豁免权。
3. **关系是结构，不是标签**
   - “喜欢/忠诚/反对”不够，关系必须体现债务、依赖、误判、权力差或路线分裂。
4. **冲突必须落地到条件**
   - 冲突要落到资源、时间、身份、信息差、代价，而不是空喊立场。
5. **行为必须留下余波**
   - 主角和配角的行动都要改动局面；“无事发生”只能是例外，不该是常态。

## 最低通过线

对一个“重要角色”，至少要满足：

- 有独立目标
- 有不能退的底线
- 有至少一个误判或盲点
- 有主角不在场时仍会发生的行动
- 离场后还能留下结构性影响

## 对 `novel-outline`

在大纲阶段，至少锁定：

- 主角的权力边界
- 核心角色分层
- 关系与阵营的最小分歧
- 核心冲突的代价与后果链
- 关键场景中谁主动、谁防守、谁误判

## 对 `novel-review`

评审时优先问：

- 谁看起来像功能件？
- 哪些阵营像一个声道？
- 主角是不是自动正确、自动有效、自动被理解？
- 有没有一整章只有推进，没有反噬？
- 有哪些角色离开主角视线就像停止运转？

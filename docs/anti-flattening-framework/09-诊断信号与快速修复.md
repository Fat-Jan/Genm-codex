# 诊断信号与快速修复

## 目的

把“看出问题”转成“如何低成本修”。

## 角色写扁信号

- 去掉标签后，这个人就不存在了
- 他只在需要时出现
- 他从不做不利于主角的正确事
- 主角不在场时，他像暂停了

## 关系写死信号

- 关系只有态度，没有结构
- 关系变化只靠一句话翻转
- 关系对剧情没有持续影响

## 阵营写平信号

- 同阵营全员一个声道
- 对立方只剩蠢和坏
- 联盟内没有成本分歧

## 快速修复动作

| 问题 | 最小修法 |
|------|----------|
| 角色像标签 | 给他一个不方便说出口的真实需求 |
| 角色像工具人 | 给他一段主角不在场也会做的行动 |
| 角色太正确 | 给他一个会推着他做错判断的盲点 |
| 关系太空 | 补一笔债务、依赖或权力差 |
| 阵营太齐 | 拆出至少两条路线和两种代价容忍度 |
| 推进太顺 | 给成功补一个残账、代价或新麻烦 |
| 章末太满 | 把“全成了”改成“成了一半，但代价暴露了” |

## 何时该走 `novel-fix`

- 问题集中在 1-2 个角色
- 关系结构能通过补线索、补动作、补代价修正
- 不需要推翻章节目标或顺序

## 何时该走 `novel-rewrite`

- 主角特权已经吞掉整章
- 冲突来源本身就是假的
- 角色动力与事件顺序不兼容
- 需要重建章功能和场景次序

## 对 `novel-review`

评审报告最好把问题落成：

- 症状
- 判断依据
- 最小修法
- 是否仍在 `fix` 范围内

## 对 `novel-fix`

优先选择“最小能让人物恢复活人感”的动作，不为局部补丁偷渡重写。

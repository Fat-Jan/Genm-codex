# 反脸谱化体系 QUICK 速查卡

## 目的

把 `反脸谱化` 体系的可执行规则压成一张速查卡，降低每次 review 时重复读取 12 份文档的成本。

它不是新的规则源，而是挂在现有：

- `core-constraints`
- Fanqie bucket 规则
- 反脸谱化结构规则

之上的一张**速查执行门**。

一句话：

- 反脸谱化负责"把人写回人"
- 本卡负责抓"人物/关系/阵营是不是写扁了"

## 自动触发范围

当满足任一条件时，这张卡应自动生效：

- 任何 `novel-review` 或 `novel-precheck` 任务
- 任何涉及角色、关系、阵营的 `novel-write` 任务
- 显式传入 `anti_flattening_check = true`

## 使用位置

### 对 `novel-review`

作为**强制第一遍漏斗**使用。

顺序固定：

1. 先判角色是否脸谱化
2. 再判关系是否结构化
3. 最后判阵营是否分歧化

如果第一层不过，先报结构失败，不要先谈文风、节奏词或修辞。

### 对 `novel-write`

作为**轻量预检卡**使用。

写作阶段不做完整审稿，但必须提前防：

- 关键角色只有标签，没有独立目标
- 关系只有态度，没有结构
- 阵营只有敌我，没有内部路线

## 漏斗逻辑

### 第一层：角色不脸谱化吗

关键角色必须同时满足 5 条：

1. 有独立目标
2. 有不能退的底线
3. 有至少一个误判或盲点
4. 有主角不在场时仍会发生的行动
5. 离场后还能留下结构性影响

如果缺任意一条，默认按 `红` 看。

#### 不算通过的情况

- 只有标签（忠诚/恶毒/聪明）
- 只在需要时出现
- 只为主角提供信息或制造爽点
- 主角不在场时像暂停了
- 离场后没有留下任何结构性影响

### 第二层：关系有结构吗

第一层过了，再看关系是不是"真结构"，不是"空态度"。

重点抓 4 件事：

1. 关系体现债务、依赖、误判、权力差或路线分裂
2. 关系变化不是靠一句话翻转
3. 关系对剧情有持续影响
4. 关系不是只有敌我标签，而是有试探、切割、债务、误判、互卡

#### 高频红灯

- 关系只有"喜欢/忠诚/反对"
- 关系变化只靠一句话翻转
- 关系对剧情没有持续影响
- 关系只有敌我标签

### 第三层：阵营有分歧吗

第二层过了，再看阵营是不是"真分歧"，不是"一个声道"。

重点抓 4 件事：

1. 同阵营不是全员一个声道
2. 对立方不是只剩蠢和坏
3. 联盟内有成本分歧
4. 阵营内至少分出两条路线和两种代价容忍度

#### 高频红灯

- 同阵营全员一个声道
- 对立方只剩蠢和坏
- 联盟内没有成本分歧
- 所有人都只围着主角转

## 判级

- `绿`
  - 三层都过
  - 本章可继续
- `黄`
  - 第一层过
  - 但第二层或第三层明显偏弱
  - 优先补核心关系或阵营场景
- `红`
  - 第一层不过
  - 或第二层严重不过
  - 本章本质是角色功能化，先重做核心人物

## 根因模板

不要把"字数不够"当根因。

优先从以下五类里归因：

- `A 类：角色脸谱化`
- `B 类：关系空洞化`
- `C 类：阵营同质化`
- `D 类：余波不足`
- `E 类：多项并发`

推荐的一句话写法：

- 这章不是单纯写短了，而是 `角色脸谱化 / 关系空洞化 / 阵营同质化`，所以人、关系、阵营一起塌了。

## `novel-review` 输出契约

当本卡生效时，review 结果应额外给出一段 `anti_flattening_summary`：

```md
anti_flattening_summary:
  funnel_grade:
  failed_layer:
  root_cause:
  most_function_like_role:
  most_structural_role:
  most_divergent_faction:
```

要求：

- `failed_layer` 只能写第一层 / 第二层 / 第三层
- `root_cause` 用上面的 `A-E` 归类
- `most_function_like_role` 必须点名最像功能件的角色；若暂无明显对象可写 `none`
- `most_structural_role` 必须点名关系结构最扎实的角色；若暂无明显对象可写 `none`
- `most_divergent_faction` 必须点名最有内部分歧的阵营；若暂无明显对象可写 `none`

## `novel-write` 轻量预检

写章前至少快速过这 5 问：

1. 这章的关键角色有独立目标吗
2. 关系有结构（债务/依赖/误判/权力差）吗
3. 阵营有内部路线分歧吗
4. 主角不在场时，关键角色会继续动作吗
5. 离场角色会留下结构性影响吗

如果前 3 问里有 2 问答不实，不要把本章直接写成正文，先回到章纲补人物骨架。

## 快速修复动作

| 问题 | 最小修法 |
|------|----------|
| 角色像标签 | 给他一个不方便说出口的真实需求 |
| 角色像工具人 | 给他一段主角不在场也会做的行动 |
| 角色太正确 | 给他一个会推着他做错判断的盲点 |
| 关系太空 | 补一笔债务、依赖或权力差 |
| 阵营太齐 | 拆出至少两条路线和两种代价容忍度 |
| 推进太顺 | 给成功补一个残账、代价或新麻烦 |
| 章末太满 | 把"全成了"改成"成了一半，但代价暴露了" |

## 何时该走 `novel-fix`

- 问题集中在 1-2 个角色
- 关系结构能通过补线索、补动作、补代价修正
- 不需要推翻章节目标或顺序

## 何时该走 `novel-rewrite`

- 主角特权已经吞掉整章
- 冲突来源本身就是假的
- 角色动力与事件顺序不兼容
- 需要重建章功能和场景次序

## 反脸谱化速查

### 五条核心原则

1. **人先于功能**：关键人物先回答"他是谁、要什么、怕什么"，再回答"他为主线做什么"
2. **主角中心可以存在，但不能垄断世界**：主角可以是故事轴心，但不能独占全部解释权和后果豁免权
3. **关系是结构，不是标签**："喜欢/忠诚/反对"不够，关系必须体现债务、依赖、误判、权力差或路线分裂
4. **冲突必须落地到条件**：冲突要落到资源、时间、身份、信息差、代价，而不是空喊立场
5. **行为必须留下余波**：主角和配角的行动都要改动局面；"无事发生"只能是例外，不该是常态

### 最低通过线

对一个"重要角色"，至少要满足：

- 有独立目标
- 有不能退的底线
- 有至少一个误判或盲点
- 有主角不在场时仍会发生的行动
- 离场后还能留下结构性影响

### 角色写扁信号

- 去掉标签后，这个人就不存在了
- 他只在需要时出现
- 他从不做不利于主角的正确事
- 主角不在场时，他像暂停了

### 关系写死信号

- 关系只有态度，没有结构
- 关系变化只靠一句话翻转
- 关系对剧情没有持续影响

### 阵营写平信号

- 同阵营全员一个声道
- 对立方只剩蠢和坏
- 联盟内没有成本分歧

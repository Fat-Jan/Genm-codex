# 检查清单与评分规约

## 目标

统一 `novel-outline` / `novel-review` 的判断口径，减少“同问题不同判”的漂移。

## `novel-outline` 检查清单

### 通过项

- 主角是否有明确权力边界
- 核心角色是否完成分层
- 关键角色是否有动力卡而不只是标签
- 关系网是否至少存在一处结构性张力
- 核心阵营是否至少分出两种路线
- 关键冲突是否落到条件和代价
- 关键章或关键场景是否指定主动方 / 防守方 / 误判点
- 当前流派高发故障是否被显式预防

### 判级建议

- `pass`
  - 无明显主角特权失衡
  - 群像结构已能支撑正文
- `warn`
  - 角色/关系/阵营有一项明显偏弱
  - 适合先补大纲再写
- `fail`
  - 核心人物仍是标签驱动
  - 冲突无代价
  - 阵营无分歧

## `novel-review` 评分维度

| 维度 | 说明 | 分值 |
|------|------|------|
| `narrative_power_balance` | 主角特权是否失衡 | 15 |
| `character_agency` | 关键角色是否有独立行动逻辑 | 20 |
| `relationship_tension` | 关系是否有结构，不是空态度 | 15 |
| `faction_divergence` | 阵营内是否存在真实分歧 | 10 |
| `conflict_and_cost` | 冲突是否有条件、阻力和代价 | 20 |
| `scene_mobility` | 场景内是否不是只有主角在动 | 10 |
| `genre_fault_control` | 是否规避本流派典型脸谱故障 | 10 |

总分：100

## 评分档位

- `85-100`
  - 通过，只有轻微提醒
- `70-84`
  - 可继续，但建议 `novel-fix` 或 `novel-polish`
- `55-69`
  - 明显失衡，优先 `novel-fix`
- `<55`
  - 结构层失败，优先 `novel-rewrite`

## 严重度

### Critical

- 主角特权直接吞掉他人人生
- 关键阵营完全无内部逻辑差异
- 关键冲突没有成本

### Warning

- 配角有一定行动，但仍明显围主角服务
- 关系有变化，但结构仍薄

### Note

- 单个功能角色略扁
- 个别场景的被动比例偏高

## 输出格式建议

```md
anti_flattening_summary:
  score:
  grade:
  critical_issues:
  warnings:
  suggested_fixes:
  recommended_next_action:
```

## 对 `novel-review`

可把上述维度写入 `dimension_scores`，不要求额外引入新 state 中心文件。

推荐的轻量写回方式：

- `chapter_meta[N].anti_flattening_flags`
- `chapter_meta[N].anti_flattening_summary`

推荐的 `dimension_scores` 附加键：

- `人物立体度`
- `关系张力`
- `阵营分歧`
- `代价感`

# 流派故障库

## 目的

通用原则之外，快速定位各流派最常见的脸谱化故障。

## 故障表

| 流派 | 高发故障 | 典型信号 | 第一修复动作 |
|------|----------|----------|--------------|
| 穿书 | 原书角色被写成剧情节点 | 配角只有“原著设定”没有现实惯性 | 补“原书之外的人生”与离场行动 |
| 穿越 | 本地人失去解释权 | 穿越者现代认知自动碾压全场 | 补本地规则、旧秩序和本地人的正确性 |
| 系统/游戏 | NPC 任务对象化 | 角色只在发任务、掉奖励、等主角救 | 补独立欲望与失败代价 |
| 修仙/升级 | 战力台阶化 | 人物只是给主角试刀、递资源、送副本 | 补宗门/家族/同门内部路线与成本 |
| 宫斗/权谋 | 立场喇叭化 | 忠臣、反派、妃嫔、家族都像一种口音 | 补同阵营内分歧与利益错位 |
| 末世/生存 | 价值观样本化 | 角色只代表“善/恶/冷酷/圣母” | 补生存账本与现实妥协 |
| 恋爱/后宫 | 偏好标签化 | 角色只剩属性、职业、外形、CP 功能 | 补非情爱目标和独立羞耻/欲望 |
| 悬疑/推理 | 线索容器化 | 证人、嫌疑人只在抛线索时存在 | 补私心、谎言成本和信息守恒 |

## 使用方式

1. 先判主流派
2. 再看混合流派的次级故障
3. 最后决定用哪一组修复动作

## 混合流派提醒

- 穿书 + 恋爱：最容易同时出现“原书残影化”和“偏好标签化”
- 系统 + 修仙：最容易同时出现“任务对象化”和“战力台阶化”
- 宫斗 + 穿越：最容易出现“本地人集体失智”

## 对 `novel-outline`

用于提前预防本流派最容易写扁的层级。

## 对 `novel-review`

用于定向指出“为什么它不是普遍意义的平，而是本流派特有的故障”。

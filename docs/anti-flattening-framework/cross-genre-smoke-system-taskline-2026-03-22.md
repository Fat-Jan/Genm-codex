日期：2026-03-22

补上此前缺失的 `系统 / 游戏任务线` 样本，验证反脸谱化体系能不能处理最典型的一类硬问题：

- 主角把别人当任务对象
- 系统把最快路径包装成唯一正确路径
- 居民 / 队友 / NPC 被压成签字机、好感包、任务节点

## 关键证据

### 结构证据

系统面板给的是：

- 三小时收满签字
- 优先拿下最易动摇目标

正文真正执行的是：

- 当众念出问题条款
- 让住户自己把各自的现实账摊上桌
- 接受“奖励缩水、任务低效”，也不把人压成签字机

### 角色证据

- 方琴不是受助 NPC，她保的是孩子学籍
- 吴建国不是单线阻碍，他是在拿补偿款救儿子的债
- 何书宁不是纸片反派，她推进签字是因为绩效和父亲手术费

这意味着：

- “他人不是任务对象”不只写在主角心里
- 还真正体现在每个人都有自己不能退的线

### 主角特权证据

陈曜有系统，但他没有被写成上帝视角：

- 他能看任务
- 但看不到住户全部真实代价
- 他能优化路径
- 但不能替别人决定什么算可承受后果

这正是系统样本最值钱的地方。

## 和前三条样本的区别

### 对比宫斗样本

宫斗样本最重要的是：

- 家门秩序
- 同盟单声道
- 主角不能吞掉家族群像

系统样本最重要的是：

- 面板逻辑不能吞掉人情逻辑
- 最优路径不能自动等于正确路径
- 居民不能退化成任务包

### 对比双女主替身样本

双女主样本看的是：

- 裂口
- 标签反夺权
- 执行人和协助人的边界

系统样本看的是：

- 人有没有被主角压成“对象”
- 系统是不是在替主角殖民他人的人生

### 对比现实情感样本

现实情感样本看的是：

- 现实代价链
- 支持者是否保自己的秩序

系统样本看的是：

- 奖励和现实后果是否对撞
- “最快完成任务”是否被主角主动拒绝

## 结论

这条样本把反脸谱化体系最硬的一类问题补上了：

1. 它不只适合人物关系戏
2. 它也适合系统、任务、游戏化叙事
3. 它能明确检查：
   - 主角有没有把他人压成任务节点
   - 系统是不是在替主角提供不受约束的解释权
   - 任务完成是否牺牲了他人的现实处境

到这里，仓库里已经有四类真实证据：

- 宫斗宅斗
- 双女主替身
- 现实情感
- 系统任务线

这足以证明：反脸谱化体系已经跨过“只适合某一类文”的阶段，进入了可跨流派复用的质量门状态。

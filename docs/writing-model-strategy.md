# Genm-codex 写作模型与思考强度策略

## 目的

这份文档不回答“平台上最新有哪些模型”，而回答更稳定、更实用的问题：

- 写小说时，哪类任务该用更强模型
- 哪类任务该用什么思考强度
- 哪些任务不该默认开到最高强度

目标是提高：

- 正文质量
- 稳定性
- 成本效率

---

## 总原则

### 1. 规划型任务优先强模型

适合：

- 书名 / 简介 / 包装
- 总纲 / 卷纲
- 复杂重写
- 大转折章
- 系统边界与 RC 计划

### 2. 产出型任务优先稳定强模型

适合：

- 日常章节写作
- 番外
- 包装文案
- 局部改写

### 3. 机械型任务不值得最高强度

适合：

- 查询
- 状态汇总
- 索引
- 轻量检查

---

## 思考强度建议

### `extra high`

只在这些场景使用：

- 卷级 / 全书级大纲设计
- 开篇第一章
- 复杂结构重写
- 多伏笔回收章
- 系统边界与 RC 计划

### `high`

默认主力档位。

适合：

- 日常章节写作
- review
- fix
- polish
- precheck
- package
- learn

### `medium`

适合：

- query
- status
- index
- retrieve
- 轻量设定查询
- 轻量日志和工作流检查

---

## 按任务类型给建议

### 1. 新书起盘

- 模型：强模型
- 强度：`high`

关键方案定稿可升到 `extra high`

### 2. 总纲 / 卷纲 / 包装

- 模型：强模型
- 强度：
  - 日常方案：`high`
  - 关键定稿：`extra high`

### 3. 日常章节写作

- 模型：强模型
- 强度：`high`

不建议默认 `extra high`

### 3.1 批量章节写作

- 模型：强模型
- 强度：`high`
- **一次性最多 3 章**

原因：

- 连续超过 3 章时，更容易从“正文章”滑成“剧情摘要”
- 质量问题通常出现在后半程，不是第一章
- 所以批量写作必须按：
  - `最多 3 章`
  - `每批后过质量门`

### 4. 审查与修复

- `review`：强模型 + `high`
- `fix`：强模型 + `high`
- `polish`：强模型 + `high`
- `rewrite`：
  - 轻量：`high`
  - 结构级：`extra high`

### 5. 投稿准备

- `precheck`：强模型 + `high`
- `export`：中高模型 + `medium` / `high`

### 6. 查询 / 状态 / 索引

- 模型：中高模型
- 强度：`medium`

只有需要复杂判断时才升到 `high`

---

## 成本与质量平衡建议

### 不要默认全流程最高强度

不推荐：

- 所有任务都用最强模型
- 所有任务都开 `extra high`

原因：

- 速度变慢
- 成本变高
- 普通章节更容易写得发紧

### 更合理的分层

#### 规划层

- 强模型
- `extra high` / `high`

#### 产出层

- 强模型
- `high`

#### 机械层

- 中高模型
- `medium`

---

## 当前推荐默认

如果没有特殊说明，当前最推荐的默认配置是：

- 系统规划 / 关键结构任务：强模型 + `extra high`
- 日常写作 / 包装 / 审查修复：强模型 + `high`
- 查询 / 状态 / 索引 / 轻量检查：中高模型 + `medium`

---

## 一句话结论

写小说不是“强度越高越好”。

真正高性价比的做法是：

- 关键结构任务上高强度
- 日常正文用稳定强模型 + `high`
- 机械任务降到 `medium`

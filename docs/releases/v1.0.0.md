# Genm-codex v1.0.0

## Summary

`v1.0.0` marks the first formal stable release of `Genm-codex`.

This release means:

- the default workflow is now considered stable
- the default-vs-experimental boundary is documented
- the sample-driven writing loop has been validated

## What `v1.0.0` includes

### Default workflow

- bootstrap
- chapter production
- review / fix / polish / rewrite routing
- packaging
- pre-submission checks
- export
- learning loop

### Default stable capability set

This release treats the following as default-stable:

- project bootstrap
- outline generation
- chapter writing
- review
- fix / polish / rewrite routing
- packaging
- precheck
- export
- learning
- status / query / retrieve / index / snapshot / resume / workflow
- character / setting / foreshadowing

## What `v1.0.0` does **not** promise

It does **not** mean:

- all experimental capabilities are now stable
- `novel-scan` is part of the default workflow
- shared sync strategy is permanently finalized

Those remain outside the formal default promise.

## Why release now

The project now has:

- a documented default workflow
- a documented stable-vs-experimental boundary
- readiness evidence
- RC planning and execution records
- no confirmed blocker preventing a stable default workflow release

## Next step after `v1.0.0`

The next phase should be:

- `v1.x` stabilization and evolution

not another migration-era phase model.

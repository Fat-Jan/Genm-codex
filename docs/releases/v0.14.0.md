# Genm-codex v0.14.0

## Summary

`v0.14.0` marks the Phase 10 checkpoint:

- packaging and market-signal convergence
- stable quality-routing signals

## Highlights

### Packaging layer convergence

`novel-package` is no longer only a packaging candidate generator.

It now has stronger judgment behavior around:

- current text-carrying capacity
- existing packaging fit
- whether packaging should be updated
- market-adjustment-aware packaging direction

### Stable quality routing

The quality loop gained more stable downstream routing:

- `novel-review` now persists `recommended_next_action`
- `novel-precheck` now emits:
  - `ready-now`
  - `revise-then-submit`
  - `do-not-submit`
  - `packaging-needs-update: yes|no`

## Validation

This checkpoint includes validated smoke coverage for:

- `novel-package`
- `novel-precheck`
- `novel-review`

plus prior Phase 9 packaging and quality-loop integration coverage.

## Practical impact

At this point, `Genm-codex` is moving beyond “migrated commands” and toward a more coherent writing-decision system:

- packaging is judged against current text
- review results can directly drive next actions
- precheck can speak to both submission readiness and packaging readiness

## Notes

This release is still pre-`1.0.0`, but it represents a stronger product boundary than the earlier migration-focused checkpoints.

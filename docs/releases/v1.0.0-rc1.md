# Genm-codex v1.0.0-rc1

## Summary

`v1.0.0-rc1` marks the first formal release-candidate checkpoint for `Genm-codex`.

This does **not** mean the project is already `v1.0.0`.

It means:

- the default workflow is defined
- the default-vs-experimental boundary is documented
- readiness evidence exists
- RC execution preparation has been completed

## Included scope

The RC scope is intentionally limited to the default workflow:

- bootstrap
- chapter production
- review / fix / polish / rewrite routing
- packaging
- pre-submission checks
- export
- learning loop

## Explicit exclusions

The following remain outside RC pass criteria:

- `novel-scan`
- shared governance enhancement commands and report modes
- docs-first help / tutorial flows
- environment guide flows such as `config / test`

## Why RC now

The project now has:

- a documented default workflow
- a documented `v1.0.0` boundary
- readiness checklist and readiness assessment
- an RC execution log
- a writing model / reasoning strategy

This is enough to begin RC without pretending the final `1.0` decision is already done.

## RC mode rules

From this point forward, the preferred posture is:

- do not expand default capability scope
- only fix blocker-class issues
- keep experimental abilities clearly marked

## Next step after RC1

The next meaningful milestone is:

- gathering blocker findings from real usage
- deciding whether the project can move from `rc1` to formal `v1.0.0`

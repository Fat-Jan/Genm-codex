# Genm-codex 第二阶段优先级

## 推荐顺序

### P0

- `novel-query`
- `novel-status`

原因：
- 它们能补足“查看状态/检索信息”的日常使用面
- 对现有核心闭环补益最大

### P1

- `novel-character`
- `novel-setting`
- `novel-foreshadowing`

原因：
- 它们是创作辅助能力
- 在核心闭环已经跑通后，价值才会充分体现

### P2

- 是否继续迁移：
  - `novel-batch`
  - `novel-query` 扩展模式
  - `novel-status` 高级统计

### 不建议短期迁移

- 已定性为历史/参考文档的命令：
  - `novel-stats`
  - `novel-outline-intelligent`
  - `novel-content-generate-polish`
  - `novel-inheritance`
  - `enrichment`

## 阶段目标

第二阶段不追求数量，目标是补齐“创作中台能力”：

- 查
- 看
- 管

而不是继续堆概念命令。

# Cross-Platform Entity Evidence Standard (`v1.5` Trial)

> Status: `active-trial-standard`
>
> 这份文档当前仍是 evidence 工作流的试行标准来源，不应在本轮归档整理中迁移出 `docs/10-进行中/`。

## 目的

这份文档定义 profile 交叉压证据包的**试行标准**。

它服务的不是“先把入口补满”，而是“让后续 agent 能稳定补出可复查、可下钻、可继续写判断的官方实体页证据”。

---

## 证据等级

### `A` 级：官方实体页

只接受可直接代表作品或内容实体的官方页面：

- `https://fanqienovel.com/page/...`
- `https://m.qidian.com/book/...`
- `https://book.qidian.com/info/...`
- `https://www.jjwxc.net/onebook.php?novelid=...`
- `https://static.jjwxc.net/onebook.php?novelid=...`
- `https://my.jjwxc.net/onebook.php?novelid=...`

用途：

- 作为最终证据包主证据
- 支撑 profile 匹配判断
- 给后续字段上升提供样本基础

### `B` 级：官方分类 / 频道 / 榜单 / 专题页

只接受官方域名下的聚合页：

- 官方分类页
- 官方频道页
- 官方榜单页
- 官方关键词页 / 专题页

用途：

- 当某个平台暂时找不到高置信作品页时，用作补充入口
- 证明该平台确实存在该题材 / bucket / 标签组织

### `C` 级：官方搜索页

例如：

- `fanqienovel.com/search/...`
- `m.qidian.com/soushu/...`
- `jjwxc.net/search.php?...`

用途：

- 仅作检索线索
- 不得作为最终证据包的完成项

---

## 完成门槛

单个 profile 只有满足下面条件，才能标记为 `complete`：

1. 番茄至少 `1 条 A 级`
2. 起点至少 `1 条 A 或 B 级`
3. 晋江至少 `1 条 A 或 B 级`
4. 起点 / 晋江里至少有一个平台为 `A 级`
5. 最终条目中不得保留 `C` 级搜索页充当完成项

推荐目标：

- 最优：`A / A / A`
- 可接受：`A / A / B` 或 `A / B / A`

不可接受：

- `A / C / C`
- `B / B / C`
- 任何把搜索页当最终交付的写法

## 条目模板

后续 agent 更新单个 profile 时，统一使用这个结构：

```md
### `<slug>`

- 状态：`complete | partial`
- 完成等级：`A/A/A`、`A/A/B` 等

### 番茄入口

- `A | <url>`
  - `<页面标题>`
  - 匹配：`<关键词 1>` / `<关键词 2>` / `<关键词 3>`

### 起点入口

- `A | <url>` 或 `B | <url>`
  - `<页面标题>`
  - 匹配：`<关键词 1>` / `<关键词 2>` / `<关键词 3>`

### 晋江入口

- `A | <url>` 或 `B | <url>`
  - `<页面标题>`
  - 匹配：`<关键词 1>` / `<关键词 2>` / `<关键词 3>`

### 备注

- 如有必要，写平台偏差或证据不足说明
```

---

## 推荐流程

1. 先用 `Exa` 搜官方域名，优先拿 `A` 级结果
2. 如果单平台 `A` 级难找，再退到 `B` 级
3. 只在搜不到实体页时临时保留 `C` 级作为线索，但不要写进最终完成项
4. 每次只补一小批高把握 profile，避免整批低置信 URL 污染文档

## 失败处理

遇到以下情况时，不要脑补：

- Exa 只给出弱相关结果
- 某平台只有搜索页，没有实体页
- 标题能对上，但题材标签对不上

统一写法：

- 状态：`partial`
- 缺口平台明确列出
- 说明“当前仅有 `C` 级线索，待补抓 `A/B` 级实体页”

这比把弱相关 URL 塞进证据包更可靠。

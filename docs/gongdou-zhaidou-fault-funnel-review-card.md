# 宫斗宅斗故障漏斗判定卡

## 目的

把 `宫斗宅斗` 这条线最常见的塌章故障收成一张可执行的 bucket 级 review 卡。

它不是新的通用规则源，而是挂在现有：

- `core-constraints`
- Fanqie bucket 规则
- 反脸谱化结构规则

之上的一张**严口径专项门**。

一句话：

- 反脸谱化负责“把人写回人”
- 本卡负责抓“看着在推进，实际没有发生有效换手”

## 自动触发范围

当满足任一条件时，这张卡应自动生效：

- `meta.platform = 番茄` 且当前 `genre_profile.bucket = 宫斗宅斗`
- 显式传入 `content_bucket = 宫斗宅斗`
- 当前 tagpack 的 `base_bucket = 宫斗宅斗`

## 使用位置

### 对 `novel-review`

作为**强制第一遍漏斗**使用。

顺序固定：

1. 先判有没有有效交易单元
2. 再判这笔交易是不是真交易
3. 最后判参与交易的人是不是活的

如果第一层不过，先报结构失败，不要先谈文风、节奏词或修辞。

### 对 `novel-write`

作为**轻量预检卡**使用。

写作阶段不做完整审稿，但必须提前防：

- 本章只有提纲推进，没有交易单元
- 一借父权 / 家法 / 身份就通
- 对手只剩放狠话
- 配角只剩送消息或递刀

## 漏斗逻辑

### 第一层：有有效交易单元吗

本章必须同时满足 3 条：

1. 有人明确想拿或想守一个具体东西
2. 过程里出现了可见阻力、反制或试探
3. 章末至少改动了一本账：
   - `权力账`
   - `信息账`
   - `关系账`
   - `资源账`

如果缺任意一条，这章默认按 `红` 看。

#### 不算通过的情况

- 只有回顾
- 只有心理活动
- 只有口头威胁
- 只有“她决定下一章继续查”
- 只有“对手不会善罢甘休”
- 只有铺气氛，但四本账一本没变

### 第二层：这笔交易真实吗

第一层过了，再看是不是“真交易”，不是“顺推”。

重点抓 4 件事：

1. 主角想拿的东西，不是靠一句话、一层身份、一位长辈就自动到手
2. 对手有真实反制，不是只负责生气和放狠话
3. 主角有可见代价，不是纯收益
4. 本章结果是 `局部赢 / 半赢 / 惨赢`，不是当章收干净

#### 宫斗宅斗高频红灯

- 借父权、家法、身份一压就通
- 对手拦得很热闹，但拦不出实际成本
- 主角赢完没有暴露、没有树敌、没有欠账
- 章末只剩“真正的风暴还在后面”

### 第三层：参与交易的人活吗

第二层过了，再看人是不是活的，不是功能件。

重点抓 4 件事：

1. 关键对手有独立算盘，不只是“针对主角”
2. 配角不是纯消息口、情绪口、工具口
3. 主角不在场时，对手仍会继续动作
4. 关系不是只有敌我标签，而是有：
   - 试探
   - 切割
   - 债务
   - 误判
   - 互卡

#### 宫斗宅斗高频红灯

- 王氏、嫡姐、老夫人、姨娘只剩“拦主角”
- 丫鬟、表妹、妈妈只负责送消息
- 所有人都只围着主角转
- 对手之间没有内部分歧，只会同声同气压主角

## 判级

- `绿`
  - 三层都过
  - 本章可继续
- `黄`
  - 第一层过
  - 但第二层或第三层明显偏弱
  - 优先补核心交易场景
- `红`
  - 第一层不过
  - 或第二层严重不过
  - 本章本质是提纲章 / 水章，先重做核心博弈

## 根因模板

不要把“字数不够”当根因。

优先从以下五类里归因：

- `A 类：无有效交易单元`
- `B 类：交易过顺`
- `C 类：对手功能化`
- `D 类：余波不足`
- `E 类：多项并发`

推荐的一句话写法：

- 这章不是单纯写短了，而是 `有效交易单元不足 / 交易过顺 / 对手功能化`，所以字数、张力、活人感一起塌了。

## `novel-review` 输出契约

当本卡生效时，review 结果应额外给出一段 `gongdou_funnel_summary`：

```md
gongdou_funnel_summary:
  funnel_grade:
  failed_layer:
  root_cause:
  changed_ledger:
  most_function_like_role:
```

要求：

- `failed_layer` 只能写第一层 / 第二层 / 第三层
- `root_cause` 用上面的 `A-E` 归类
- `changed_ledger` 至少指出一本被改动的账；若第一层失败则写 `none`
- `most_function_like_role` 必须点名最像功能件的角色；若暂无明显对象可写 `none`

## `novel-write` 轻量预检

写章前至少快速过这 5 问：

1. 这章具体在争什么
2. 谁在拦，怎么拦
3. 章末哪一本账会变
4. 主角这次要付什么
5. 哪个非主角角色带着独立算盘上场

如果前 3 问里有 2 问答不实，不要把本章直接写成正文，先回到章纲补交易骨架。

- 分页：`page_index=0..3`，`page_count=15`
- 实际抓取结果：**55 篇图文**

来源页：

- 教程中心：<https://fanqienovel.com/writer/zone/tutorial?enter_from=menu&tab=3>

## 先说结论

这 55 篇图文里，真正对当前项目最有价值的，不是零散技巧，而是 6 类可转成系统约束的写法规则：

1. **标题 / 简介规则**
2. **开头与黄金三章规则**
3. **人物“活人感”规则**
4. **剧情稳定与悬念交替规则**
5. **长线更新与存稿规则**
6. **版权 / 合规避坑规则**

也就是说，它们最适合接入的层不是“单句润色”，而是：

- `novel-package`
- `novel-outline`
- `novel-write`
- `novel-review`
- `novel-precheck`

## 一、标题 / 简介

高价值参考：

- [还不会起书名吗？这几招让你的作品脱颖而出！](https://fanqienovel.com/writer/zone/article/7186865162572267580)
- [如何起作品名称和写作品简介](https://fanqienovel.com/writer/zone/article/7025876380030074888)

提炼出的可执行规则：

- 标题必须优先服务：
  - 人物关系
  - 冲突爆点
  - 清晰可懂
- 不要为了文艺感牺牲点击理解
- 简介应更偏：
  1. 被害 / 压制
  2. 当前困局
  3. 女主或主角的反击动作
  4. 更大的后果或局势

对项目的帮助：

- `novel-package` 不能只产“好听标题”
- 需要增加：
  - 关系词检查
  - 爆点词检查
  - 共情词检查
  - 题文一致性检查

## 二、开头与黄金三章

高价值参考：

- [如何写好开篇](https://fanqienovel.com/writer/zone/article/7025879668578320398)
- [如何稳定剧情，让读者追更不停？](https://fanqienovel.com/writer/zone/article/7605818896267870270)
- `【如何写好新书】` 系列的故事支点 / 三幕式 / 多线叙事文章

提炼出的可执行规则：

- 第一屏必须尽快给“异常 / 压制 / 痛点 / 高概念”
- 第 1 章不能只是背景介绍
- 前三章必须有一次真正兑现，而不是只有更大的承诺
- 大悬念结束前，下一条悬念必须提前埋好

对项目的帮助：

- `novel-outline` 后面应显式检查：
  - 第1章第一页是否真有钩子
  - 第2章是否承接而不掉速
  - 第3章是否有第一次硬兑现
- `novel-precheck` 后面应增加：
  - 开头 100-200 字钩子检查
  - 黄金三章兑现检查

## 三、人物“活人感”

高价值参考：

- [人物立不住？4个技巧，让你笔下的人物自带“活人感”](https://fanqienovel.com/writer/zone/article/7612544314719076376)
- [人物塑造｜从模型搭建到记忆点强化](https://fanqienovel.com/writer/zone/article/7496817783338958872)
- [人物塑造｜CP塑造与避免人设崩塌的方法](https://fanqienovel.com/writer/zone/article/7502291494963773502)
- [人物塑造｜用反差标签法打造鲜活角色团](https://fanqienovel.com/writer/zone/article/7506141476644978712)
- [如何塑造人物](https://fanqienovel.com/writer/zone/article/7025880344582684703)

提炼出的可执行规则：

- 人物不能只有标签，要有：
  - 过往
  - 细节动作
  - 成长路径
  - 反差点
- 回忆不能乱加，必须服务当前行为逻辑
- 配角不能只有工具功能，要有最少一层自己的动机或偏向

对项目的帮助：

- `novel-review` 后面应增加：
  - 是否只有标签、没有行为逻辑
  - 是否只有对话、没有状态细节
  - 成长是否一蹴而就
- `novel-write` 在人物驱动桶里，要更强调：
  - 过往痕迹
  - 动作细节
  - 反差标签

## 四、剧情稳定与悬念交替

高价值参考：

- [如何稳定剧情，让读者追更不停？](https://fanqienovel.com/writer/zone/article/7605818896267870270)
- [稳定更新｜如何稳住剧情节奏](https://fanqienovel.com/writer/zone/article/7448929829954535448)
- [编辑说 | 崩文、差评、视角混乱！编辑教你如何解决！](https://fanqienovel.com/writer/zone/article/7579836380545564734)
- [小说情节如何制造矛盾冲突](https://fanqienovel.com/writer/zone/article/7025881468408692749)
- [网络小说的爽点](https://fanqienovel.com/writer/zone/article/7025881902951170055)
- [网络小说的代入感](https://fanqienovel.com/writer/zone/article/7025882180156915748)

提炼出的可执行规则：

- 悬念不能等上一个快结束了再埋下一个
- 换地图时，要强调故事目标，不要强调风景和环境
- 卡文时，不要水文，要：
  - 复盘大纲细纲
  - 直接加快进度
  - 换支线/换地图
  - 调动配角线
- 冲突要具体，爽点要可见，不是空喊

对项目的帮助：

- `novel-analyze` 后面应更明确：
  - 悬念交替是不是太晚
  - 换地图有没有割裂
  - 是否进入水文区
- `novel-review` 可增加：
  - 悬念接力
  - 换地图丝滑度
  - 爽点兑现密度

## 五、长线更新与存稿

高价值参考：

- [稳定更新｜告别卡文！四步突破十万字大关](https://fanqienovel.com/writer/zone/article/7441438066205523993)
- [如何保持稳定的更新速度](https://fanqienovel.com/writer/zone/article/7025882542381203487)
- [写网络小说需要存稿吗](https://fanqienovel.com/writer/zone/article/7025882746736082980)

提炼出的可执行规则：

- 卡文不是正常借口，卡了就回：
  - 大纲
  - 细纲
  - 配角线
  - 地图切换
- 长线写作不是靠灵感，而是靠：
  - 大纲
  - 细纲
  - 存稿
  - 固定更新纪律

对项目的帮助：

- `novel-batch`
- `novel-resume`
- `novel-workflow`

后面都可以更明确地支持：

- 存稿批次
- 卡文回退路径
- 细纲复盘入口

## 六、版权与合规

高价值参考：

- 13 篇 `【版权说】` 系列

这些不直接帮助“写得更爽”，但对真正要发出去非常重要：

- 书名和封面侵权
- 人格权侵权
- 自己抄自己
- 抄袭申诉路径
- 请假姿势

对项目的帮助：

- `novel-precheck` 后面可以加一层轻量版权提醒
- `novel-package` 后面可加：
  - 书名风险提示

## 对当前三条女频线的直接帮助

### `恶女 x 宫斗宅斗`

帮助最大。

后续重点应继续强调：

- 第一屏压制关系
- 第三章前硬兑现
- 每章章末更大局
- 反咬要合规矩，不是纯发疯

### `恶女 x 现实情感`

帮助也非常大。

后续重点应继续强调：

- 痛点前置
- 先证据，后情绪
- 现实切割动作可见
- 不要滑回“只吵架不切账”

### `恶女 x 现言甜宠`

帮助有，但更像“把它拉回可读”。

后续重点应继续强调：

- 关系绑定
- 身份反差
- 公开站位
- 糖感和危险感同时成立

而不是只靠“恶女感”硬撑。

1. 给 `novel-package` 增加标题/简介点击结构检查

3. 给 `novel-review` 增加悬念交替与换地图丝滑度判断
4. 给 `novel-precheck` 增加轻量版权与标题风险提醒

## 一句话结论

番茄作者专区的“写作技巧”图文，真正值钱的地方不是零散技巧，而是：

- **它能把“番茄写法”转成一套项目内可消费的规则。**

也就是说，它对当前项目的帮助非常直接，尤其是在：

- 包装
- 开篇
- 黄金三章
- 悬念接力
- 角色活人感
- 长线更新

这些层面。

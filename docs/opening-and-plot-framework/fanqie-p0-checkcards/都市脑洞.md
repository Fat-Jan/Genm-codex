# 都市脑洞专项检查卡

## 自动触发范围

- `meta.platform = 番茄` 且当前 `genre_profile.bucket = 都市脑洞`
- 显式传入 `content_bucket = 都市脑洞`

## 对 `novel-review`

- 先看脑洞机制是否快速见效
- 先看现实层反馈是否可见
- 如果机制只是讲得新鲜，却没有真实剧情交换，优先报结构空转

## 对 `novel-precheck`

- 优先拦机制点亮后仍迟迟不改变现实处境
- 优先拦第三章前没有形成小爽点循环
- 优先拦只有网上震惊、没有现实后果

## 核心检查问题

- 机制是否在前三章内产生可见现实收益
- 收益是否伴随试错、边界或代价
- 现实层面的职业、社交、资源位置是否发生变化
- 章末是否挂上更大一轮爽点钩子

## 红灯判定

- 只有设定新鲜，没有剧情交换
- 机制像万能许愿机，没有边界
- 第三章前没有一次完整的小爽点闭环
- 现实反馈只停留在口头震惊

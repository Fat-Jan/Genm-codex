# 开篇故障与修正

## 目标

统一识别开篇常见故障，避免每次 review 只会说“开头有点慢”。

## 常见故障

### 1. 背景先行

表现：

- 先讲世界、设定、家谱、系统说明
- 真正发生的事来得太晚

修正：

- 把解释压后
- 先让角色被一件事逼着动

### 2. 只有压迫，没有交换

表现：

- 主角一直惨
- 读者一直知道她惨
- 但前几章没有任何局势变化

修正：

- 尽快补一次小反击、误导、换账或信息跃迁

### 3. 承诺过大，近期兑现缺失

表现：

- 简介和首章承诺很大
- 但前三章都没有第一轮兑现

修正：

- 把第一次兑现做小，但必须做实

### 4. 只有设定，没有人物情绪

表现：

- 能看懂局面
- 但读者不知道该替谁急、替谁恨、替谁等

修正：

- 给人物身体反应、即时选择、关系牵动

### 5. 章末无钩

表现：

- 这一章写完了
- 但没有留下下一步想看的账

修正：

- 留一个未解问题
- 或留一个局势变更
- 或留一个刚赢就更危险的结果

## 修正优先级

1. 先修“有没有故事”
2. 再修“有没有交换”
3. 再修“有没有情绪入口”
4. 最后才修语言细节

## 对 skill 的影响

- `novel-write` 需要在前三章尤其防这几种故障
- `novel-review` 需要把故障映射成可执行问题簇
- `novel-package` 需要避免包装承诺大于正文能兑现的强度

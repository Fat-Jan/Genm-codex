# 番茄 P0 特化卡

这组文档是 `opening-and-plot-framework` 的第一批题材特化 overlay。

它们只解决一个问题：

- 当项目已经明确是番茄路线，并且当前 bucket 命中官方近年反复扶持的重点分类时，开篇方法、剧情层次、推进残账应该怎么进一步收紧。

## 当前范围

P0 共 8 类：

1. [宫斗宅斗](./宫斗宅斗.md)
2. [职场婚恋](./职场婚恋.md)
3. [青春甜宠](./青春甜宠.md)
4. [豪门总裁](./豪门总裁.md)
5. [都市日常](./都市日常.md)
6. [玄幻脑洞](./玄幻脑洞.md)
7. [都市脑洞](./都市脑洞.md)
8. [历史脑洞](./历史脑洞.md)

## 使用方式

- `novel-outline`：命中 bucket 时，用来锁开篇 promise、前三章兑现和线层次
- `novel-write`：命中 bucket 时，用来防典型塌章
- `novel-review` / `novel-precheck`：命中 bucket 时，用来做专项故障筛查
- `novel-package`：命中 bucket 时，用来约束标题/简介 promise 不要跑飞

对应的 `review / precheck` 专项门入口：

- [番茄 P0 专项检查卡](../fanqie-p0-checkcards/README.md)

## 原则

- overlay 只加强，不替代通用骨架
- overlay 不能覆盖 canon、bucket 硬约束或已冻结真值
- overlay 应优先复用现有 profile / bucket / 质量门，而不是再造一套平行名词

## 当前统一结构

这 8 张卡现在都按同一结构维护：

1. `开篇公式`
2. `黄金三章最低要求`
3. `剧情层次重点`
4. `高频故障清单`
5. `precheck 专项检查点`

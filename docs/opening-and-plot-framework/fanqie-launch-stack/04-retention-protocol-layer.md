# Retention Protocol Layer

## 目标

把番茄平台级“留人/追读”规则单独抽出来，不藏进任何单张语法卡。

## 第一版默认协议

1. `first-page-hook`
2. `chapter-end-hook`
3. `hook-handover`
4. `first-hard-payoff`
5. `half-win-with-residual`
6. `no-foolish-humiliation`
7. `early-stability`

## 规则解释

### `first-page-hook`

- 首页尽快给异常、压制、错位、风险或强欲望

### `chapter-end-hook`

- 每章末尾尽量留下继续读下一章的轻钩

### `hook-handover`

- 大悬念未结束前，下一条悬念应提前接力

### `first-hard-payoff`

- 前三章至少一次真正换账，不只放大 promise

### `half-win-with-residual`

- 第一次赢最好带代价、残账或更深风险

### `no-foolish-humiliation`

- 不靠主角低级失误硬造压制或憋屈

### `early-stability`

- 前十万字之前的主推进线不能散

## 推荐输出

- `retention_protocol.enabled_rules`
- `retention_protocol.priority_rules`
- `retention_protocol.violations`

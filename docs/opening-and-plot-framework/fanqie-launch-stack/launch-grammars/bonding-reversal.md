# bonding-reversal

## best_for

- 婚恋
- 豪门/职场绑定
- 同居、合约、误会、站位逆转

## not_for

- 主要快感来自规则探索或副本试错

## reader_promise

- 看关系先卡死，再在互相利用里发生逆转

## chapter_1_3_minimums

- 第 1 章明确绑定条件
- 第 2 章出现第一次关系试探或错位
- 第 3 章出现一次站位变化或利益交换

## scene_engine

- `Goal -> Conflict -> Outcome/Disaster -> Reaction -> Dilemma -> Decision`

## review_watchpoints

- 绑定是否过虚
- 关系推进是否只靠嘴说
- 站位变化是否有现实后果

## precheck_failures

- 关系在走，主冲突没动
- 包装 promise 超过正文兑现

## package_guardrails

- 不把后期完全站队写成前三章已兑现事实

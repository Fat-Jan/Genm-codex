# investigation-reveal

## best_for

- 案件
- 旧案翻新
- 身份谜团
- 内幕追查

## not_for

- 主要卖点是纯甜宠日常或升级修炼

## reader_promise

- 看主角抓住信息差，一层层翻页

## chapter_1_3_minimums

- 第 1 章抛核心疑点
- 第 2 章出现第一轮追查和误判
- 第 3 章出现第一次有效翻页

## scene_engine

- `Inciting Incident -> Progressive Complications -> Crisis -> Climax -> Resolution`

## review_watchpoints

- 疑点是否足够具体
- 追查是否有阻力
- 翻页是否真改变理解

## precheck_failures

- 一直埋疑点不翻页
- 追查只靠巧合推进

## package_guardrails

- 不把最终真相当成前三章已交付卖点

# rule-trial

## best_for

- 规则怪谈
- 异能脑洞
- 系统规则亮相
- 穿书/奇异机制试错

## not_for

- 主要卖点是现实关系盘活

## reader_promise

- 看异常规则先亮相，再用试错证明代价

## chapter_1_3_minimums

- 第 1 章亮规则或异常
- 第 2 章出现第一次试错
- 第 3 章出现一次代价升级或规则反噬

## scene_engine

- `Inciting Incident -> Progressive Complications -> Crisis -> Climax -> Resolution`

## review_watchpoints

- 规则是否只讲不演
- 试错是否没有成本
- 悬念是否只藏不接

## precheck_failures

- 前三章只展示设定，没有真正换账
- 规则看起来无边界

## package_guardrails

- 不把未验证规则吹成无敌外挂

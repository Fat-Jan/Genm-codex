# resource-climb

## best_for

- 玄幻/修仙
- 经营
- 职业成长
- 资源争夺与阶段升级

## not_for

- 主要卖点是关系互卡或真相追查

## reader_promise

- 看主角先缺资源，再通过争夺、交易或试炼往上爬

## chapter_1_3_minimums

- 第 1 章锁资源缺口
- 第 2 章出现第一次争夺或门槛
- 第 3 章出现阶段收益并留下新账

## scene_engine

- `Goal -> Conflict -> Outcome/Disaster -> Reaction -> Dilemma -> Decision`

## review_watchpoints

- 升级是否太白拿
- 收益是否没有成本
- 主线是否被设定讲解拖慢

## precheck_failures

- 只有大 promise，没有阶段兑现
- 资源线薄，爽点不具体

## package_guardrails

- 不把未来大跨级收益写成前三章已兑现事实

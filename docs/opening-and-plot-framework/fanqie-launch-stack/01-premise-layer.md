# Premise Layer

## 目标

在起盘阶段先锁一句能被系统消费的一句话故事，而不是先堆设定名词。

## 这层回答什么

- 主角当前是什么负位置
- 主要想要什么
- 为什么必须动
- 这条线的即时 promise 是什么

## 推荐输出

- `premise_line`
- `premise_keywords`
- `promise_axis`

## 合格标准

### `pass`

- 读一句就知道主冲突方向
- 读一句就知道这书大概要爽在哪
- 能直接支撑标题、总纲和前三章设计

### `warn`

- 有冲突，但欲望或行动方向还虚
- 有设定，但 promise 还不够具象

### `fail`

- 只有世界观，没有人物行动
- 只有题材标签，没有可执行冲突

## 第一版约束

- 只锁起盘用压缩句
- 不把 marketing 文案当 premise
- 不在这一层讨论完整大纲

# 剧情层次模型

## 目标

让“这章发生了什么”和“这章推进了哪条线”成为两回事。

## 五类常见层次

### 1. 主线

全书最主要的冲突方向和兑现方向。

### 2. 支线

服务主线，但短期可以独立推进的局部事件链。

### 3. 人物线

角色自己的动机、成长、崩塌、选择变化。

### 4. 关系线

人与人之间的联盟、背刺、试探、亏欠、债务变化。

### 5. 世界/规则线

世界观、制度、门槛、系统限制、家族法统、权力秩序的变化。

## 使用方法

### 对总纲

至少要明确：

- 主线是什么
- 支线怎么服务主线
- 哪些人物线会反过来改变主线
- 哪些关系线不是装饰，而是真会换账

### 对章纲

至少要明确：

- 这一章主推进线是什么
- 这一章触动了哪条次级线
- 哪条线只是铺垫，哪条线真的发生了位移

## 判定标准

### 层次清晰

- 读完能说清主推进线
- 读完能知道附带触动了什么
- 不是所有变化都堆在一条线里

### 层次混乱

- 事情很多，但分不清主次
- 关系变化像顺手带过
- 规则变化和人物选择不接

## 默认提醒

- 主线不是“最重要的一件事”，而是“最持续改变读者期待的那条线”
- 支线不是水文区，而是用来丰富压力结构和兑现方式
- 好的层次不是线多，而是知道每条线什么时候该动、动多少

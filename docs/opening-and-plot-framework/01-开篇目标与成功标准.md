# 开篇目标与成功标准

## 目标

把“开篇”从背景介绍区，拉回成真正的故事入口。

## 开篇至少要完成什么

### 1. 主承诺可见

读者应在较短范围内看见：

- 这书主要会爽在哪里
- 主要冲突是什么方向
- 主角将要面对什么局

### 2. 冲突前置

开篇不要求马上打满，但必须尽快出现：

- 压迫
- 异常
- 错位
- 风险
- 选择

至少一种。

### 3. 情绪入口成立

读者需要一个进入口：

- 替主角憋屈
- 替主角着急
- 替主角好奇
- 替主角想看反击

### 4. 未解问题成立

开篇最好尽快留下至少一个可追问的问题：

- 为什么会这样
- 背后是谁
- 主角能不能翻盘
- 这次赢了之后会不会更危险

### 5. 近期兑现路径存在

尤其在前 1-3 章，不能只有大承诺。

必须能看见：

- 第一轮冲突
- 第一轮回应
- 第一轮半兑现或硬兑现

## 成功标准

### `pass`

- 首屏或首章较快进入故事状态
- 读者能分辨主承诺和当前冲突
- 前三章内存在一次明确兑现

### `warn`

- 承诺存在，但进入过慢
- 有冲突，但近期兑现路径不清
- 情绪入口较弱

### `fail`

- 大段背景先行，故事后到
- 首章基本没有能推动继续阅读的未解问题
- 前三章只有承诺，没有实际换账

## 对默认工作流的意义

- `novel-outline` 需要锁“开篇承诺 + 前三章兑现路径”
- `novel-write` 需要防止首章写成设定说明
- `novel-review` / `novel-precheck` 需要把开篇抓力当成硬检查项之一
- `novel-package` 的包装承诺不能脱离这里定义的正文承载范围

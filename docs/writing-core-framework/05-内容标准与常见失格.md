# 内容标准与常见失格

## 目标

把“这一章为什么读着不行”压成可执行、可路由的失败类型，而不是抽象差评。

## v1 内容标准

### 1. 看得懂

- 场景关系明确
- 谁在对谁做什么明确
- 章内主要推进明确

### 2. 看得下去

- 开头不被背景说明吞掉
- 中段不滑入总结腔
- 章末有真实后续驱动

### 3. 看得到变化

- 信息差有变化
- 关系有变化
- 代价或残账有变化

### 4. 承诺接得住

- 当前正文能支撑包装 promise
- 前三章能支撑开篇 promise

## 常见失格类型

### 背景先行

- 读者先收到设定说明，后收到故事入口

### 悬浮对白

- 角色一直在说，但没有动作、压迫、试探和后果

### 提纲代替演出

- 用概述交代本该在场上完成的冲突或交换

### 白拿推进

- 关键证据、同盟、资源、认可来得太省

### 假紧张

- 说危险在升级，但页面上的信息、关系、代价账本没变

### 过度承诺

- 简介、标题、开篇包装比正文近期兑现能力强太多

## 路由建议

- 单场景执行失格，优先 `novel-fix`
- 章内多处同类失格，但结构未坏，可 `novel-polish` 或 `novel-fix`
- 主 promise、主要冲突源或主要推进线写歪，优先 `novel-rewrite`

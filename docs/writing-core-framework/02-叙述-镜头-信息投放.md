# 叙述、镜头与信息投放

## 目标

控制正文的“看见方式”和“知道方式”，避免一开写就滑回背景说明和作者解说。

## 核心规则

### 1. 场景先落地，再讲解释

- 先让读者知道：
  - 人在哪
  - 正在做什么
  - 谁在对谁施压或交换
- 再决定要不要补解释

### 2. 视角稳定优先于信息完整

- 单个段落里不要随意切换“谁知道 / 谁在判断 / 谁在解释”
- 当下章回只要能支撑当前冲突，就不必一次性交代全部背景

### 3. 信息投放要服务当前冲突

允许补信息，但必须至少服务其一：

- 当前误判
- 当前选择
- 当前交换
- 当前后果

### 4. 解释一旦盖过现场，就是拖速

典型失格：

- 刚进入场景就回顾完整设定
- 对话前先长段讲关系史
- 刚出现异常就先解释世界规则

## 对下游技能的可执行影响

### `novel-write`

- 开场先锁场景锚点
- 避免“解释先于事件”

### `novel-review`

- 检查是否存在背景先行、镜头漂移、信息拥堵

### `novel-precheck`

- 检查首屏是否被解释段吞掉
- 检查前三章是否因信息投放过重而削弱 promise

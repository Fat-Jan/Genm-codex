# 剧情层次与多线编排接口

## 目的

这里不重复发明剧情层次理论，而是把当前项目里已经稳定的结构规则，转成下游技能可直接消费的接口。

主来源：

- [opening-and-plot-framework/04-剧情层次模型.md](../opening-and-plot-framework/04-剧情层次模型.md)
- [opening-and-plot-framework/05-推进链与残账设计.md](../opening-and-plot-framework/05-推进链与残账设计.md)

## 下游最小消费单元

### 1. 主推进线

当前章必须知道：

- 这一章主要推进哪一层
- 这一层的推进结果是否可见

### 2. 被触动的次级线

当前章至少要回答：

- 还有哪条关系线 / 人物线 / 规则线被触动
- 触动是正向、负向还是暂存

### 3. 残账

当前章推进后，至少保留一种可追踪残账：

- 关系残账
- 代价残账
- 信息残账
- 规则残账

### 4. 多线提醒

多线不是同时平推所有线，而是提醒：

- 哪条线在本章主导
- 哪条线只做轻触
- 哪条线不该被误写成“已经解决”

## 对技能的具体要求

### `novel-outline`

- 章纲里显式区分主线和被触动线

### `novel-write`

- 正文里让主线推进和次线触动都能看见

### `novel-review`

- 检查是不是只发生了事件，却没有线层变化

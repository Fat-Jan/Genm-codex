# 精品审核与投稿前判断

## 目标

把“能不能继续写 / 能不能拿去投 / 包装是不是该收紧”做成更稳定的判断层。

## 三档判断

### 1. 可继续追读

至少满足：

- 开头 promise 清楚
- 当前章有真实变化
- 章末有后续驱动

### 2. 可投稿

除了可追读，还应满足：

- 开篇与黄金三章没有明显失格
- 当前包装没有明显过度承诺
- review 结果没有留下结构级红灯

### 3. 应先收口再投

出现任一情况就不该直接投：

- 包装 promise 明显大于正文近期承载
- 开头抓力主要靠标题、不是靠正文
- 前三章仍以解释、铺垫、概述为主
- review 已指出结构级失败仍未修

## 对工作流的影响

### `novel-review`

- 要把发现收成少量、可执行的故障簇
- 路由到 `fix / polish / rewrite`

### `novel-precheck`

- 要输出：
  - `投稿建议`
  - `packaging-needs-update`

### `novel-package`

- 不能只做外层文案生成
- 必须读当前正文承载状态

## 对包装的直接约束

- 如果前三章还没有近端兑现，包装不能把 promise 拉得更满
- 如果当前卖点只能靠设定解释成立，包装不能当成已兑现卖点输出
- 如果关系词、身份词、权力词还没被正文真值支撑，包装应降级为暂定方向

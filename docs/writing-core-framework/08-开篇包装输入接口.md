# 开篇包装输入接口

## 目标

把“开篇方法 / 类型化开篇差异 / 精品审核标准”转成 `novel-package` 和 `novel-precheck` 可直接消费的输入。

## package 可消费字段

### 1. opening method cue

回答：

- 开篇 promise 靠什么成立
- 首屏应该前置哪种交易、压迫、异常或关系绑定

### 2. genre-difference cue

回答：

- 当前题材的开篇更偏：
  - 压迫反击
  - 关系绑定
  - 异常揭示
  - 规则试跑

### 3. premium-review cue

回答：

- 当前文本距离“可投稿”还差哪一个关键补强点

### 4. overpromise warning

回答：

- 哪类标题、简介或开篇包装说法当前不能用

### 5. payoff timing reminder

回答：

- 包装里承诺的看点，最迟应在哪个近端节点给第一次兑现

## 输出约束

### 对 `novel-package`

- 书名、简介、开篇包装必须绑定到当前正文能承载的 promise
- 如果 promise 只能在中后段兑现，就不能伪装成前三章已兑现卖点

### 对 `novel-precheck`

- 当包装 promise 与正文承载脱节时，优先打 `packaging-needs-update: yes`

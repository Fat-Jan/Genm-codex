# 写作基本功总纲

## 目标

这里的“基本功”不是文学课评语，而是默认工作流里最常见、最能直接决定可读性的正文执行质量。

它至少回答四个问题：

1. 读者能不能快速看懂此刻发生了什么
2. 场景是不是在推进，而不是在概述
3. 人物是不是通过行为、对白和后果站住了
4. 这章的句段节奏是否支撑追读，而不是拖慢

## 基本判断

- 基本功的核心不是“辞藻更漂亮”，而是**执行更可读**
- 它应该同时进入：
  - `novel-write`
  - `novel-review`
  - `novel-precheck`
- 它不应该被做成一套独立大数据库

## v1 必查项

- 场景落点是否明确
- 叙述视角是否稳定
- 信息投放是否过量
- 对白是否只在解释
- 动作是否能承接关系与冲突
- 情绪是否只停在标签
- 段落节奏是否滑向提纲化

## 和其他框架的关系

- `opening-and-plot-framework` 管结构骨架和推进账本
- `anti-flattening-framework` 管人物、关系、阵营和代价链
- 本文档管正文执行是否把这些东西**写在页面上**

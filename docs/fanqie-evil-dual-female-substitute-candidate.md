## 为什么它值得做第四条

### 1. 它天然吃“关系局”

这条线最强的不是世界观，而是：

- 谁才是真正被爱的那个人
- 谁占的是替身位
- 谁是被剧情默认会输的女配
- 谁能把叙事主位从“白月光模板”里抢回来

这很适合番茄女频的：

- 强关系词
- 强误认
- 强反转
- 强持续拉扯

### 2. 它比普通甜宠更容易做持续付费钩子

它的持续钩子不只来自男主，而来自多线关系错位：

- 女配视角翻主位
- 白月光位置被拆穿
- 替身关系转成反制关系
- 双核心女性角色持续拉扯

也就是说，它天然更适合：

- 章节末留关系悬念
- 不是只靠单点打脸

### 3. 它也比前三条更容易写崩

主要风险不是热度，而是结构控制：

- 很容易写成标签拼贴
- 很容易只剩雌竞口水战
- 很容易把“双女主”写成两个符号，不是真正双核心
- 很容易把“替身白月光”写成全靠男主回头

这也是它现在只能排第四的原因。

## 当前建议定位

最合理的定位不是：

- `恶女 x 现言甜宠` 的直接替代

而是：

- **现言甜宠主桶下的关系实验线**
- 必要时吸收 `现言脑洞` 的叙事机制
- 用现有 `substitute` profile 做上游风味参考

更具体地说：

- `base_bucket`：
  - 当前先落 `现言甜宠`
- `secondary flavor`：
  - 替身 / 白月光 / 女配反转
- `relation mode`：
  - 双核心女性关系驱动

这里的“双女主”默认解释为：

- **双核心女性关系驱动**

不是默认声明：

- 百合主线
- 纯女性向 CP 线

后者需要单独再建模，当前先不要偷换。

## 当前不建议的做法

- 不要一上来把 4 个词全塞进标题主方案
- 不要把它直接升格为第四条正式生产模板
- 不要在没有正文样本的情况下，声称它“付费率已经证实很高”
- 不要把双女主写成单纯互撕
- 不要把替身线写成“男主后悔 = 一切成立”

# Genm-codex `v1.2` Roadmap

## 目的

这份 roadmap 承接 `v1.1` 完成后的下一阶段工作。

它回答：

- `v1.1` 收口后，最值得继续推进的方向是什么
- 哪些是主线增强，哪些只是可选扩展
- 后续工作如何避免再次把过多内容塞回 `v1.1`

## 状态说明

- `[done]`：已完成
- `[in_progress]`：进行中
- `[planned]`：已确认要做
- `[blocked]`：受前置条件阻塞
- `[deferred]`：明确延后

## 当前总判断

`v1.2` 不应该继续做“大量能力补线”。

`v1.1` 已经完成了：

- Profile Contract
- shared governance
- workflow tail closure
- content-positioning
- 当前有效文档真相层

因此 `v1.2` 更应该聚焦：

1. **扩大映射覆盖面**
2. **提升内容层质量**
3. **补真实样本 smoke**
4. **决定哪些实验能力值得继续正式化**

## 主线 A：扩大 `content-positioning` 覆盖

### A1. 扩大 profile / platform 映射覆盖

- Status: `[planned]`
- 目标：
  - 让 `content-positioning-map-v1.json` 不只覆盖第一批 profile
- 优先对象：
  - `urban-brainhole`
  - `xiuxian`
  - `workplace-romance`
  - `sweet-romance`
  - `historical-brainhole`

### A2. 为更多 profile 补 `platform_positioning`

- Status: `[planned]`
- 目标：
  - 把 `fanqie` 侧的：
    - `primary_bucket`
    - `strong_tags`
    - `narrative_modes`
    - `tone_guardrails`
    - `package_cues`
  继续下沉到更多 profile 文件

## 主线 B：内容层质量升级

### B1. profile 内容字段继续校准

- Status: `[planned]`
- 目标：
  - 在已统一 contract 前提下，逐步清理和增强：
    - pacing
    - reader_expectations
    - taboos
    - platform_positioning

### B2. 清理 legacy embedded reference

- Status: `[planned]`
- 目标：
  - 逐步把 raw profile 里混入的大段：
    - `dialogue_templates`
    - `scene_description`
    - 长示例文本
  迁移到更合适的 reference 层

## 主线 C：真实样本与 smoke

### C1. 组合题材 smoke

- Status: `[planned]`
- 目标：
  - 验证 `content-positioning` 在真实项目中的：
    - outline overlay
    - write overlay
    - review lens
    - package cue

### C2. bucket / tagpack / narrative_modes 交叉样本

- Status: `[planned]`
- 目标：
  - 至少做 2-3 条跨题材 / 跨风味样本
  - 看当前 contract 是否足够稳定

## 主线 D：实验能力去留判断

### D1. `novel-scan` 是否正式化

- Status: `[deferred]`
- 目标：
  - 在已有 `v1.1` 边界基础上，重新判断：
    - 保持实验态
    - 还是进入下一阶段更稳定的候选链

### D2. `novel-log` 是否继续保留在默认主线

- Status: `[planned]`
- 目标：
  - 结合 trace log 的真实使用频率决定：
    - 保留默认主线
    - 还是降回专家能力

## 暂不做

- 不重开多 profile merge runtime
- 不做多主桶并列引擎
- 不引入 monolithic runtime / plugin system
- 不把 MCP memory 升成项目真值层

## 推荐顺序

1. 主线 A：扩大映射覆盖
2. 主线 B：内容层质量升级
3. 主线 C：真实样本 smoke
4. 主线 D：实验能力去留判断

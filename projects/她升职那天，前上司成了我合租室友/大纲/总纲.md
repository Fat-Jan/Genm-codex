# 《她升职那天，前上司成了我合租室友》总纲

```yaml
书名: 她升职那天，前上司成了我合租室友
题材: 现代言情 / 职场婚恋
目标平台: 番茄
预计章节数: 12
预计字数: 48000
内容桶: 职场婚恋
```

## 一句话简介

姜栀升职当天，接下的是一个随时会砸掉她职位的大客户；回到新租的房子，她发现新室友竟是刚跳去甲方的前上司裴述。一个掌握续约生死，一个连退房都退不起，他们只能在工作链路冲突和同屋檐下的生活摩擦里，一边算项目账，一边算关系账。

## 开篇承诺

- 第一章直接把“升职接锅 + 合租揭面”双困局砸下来。
- 前三章完成一次职业层面的小闭环：从接锅、被质疑，到拿下试运行窗口。
- 感情线不走脱离业务的空糖，所有关系推进都要经过真实工作利害和生活边界摩擦。

## 核心卖点

- 事业线和感情线互相牵制，不存在“恋爱归恋爱、工作归工作”的安全区。
- 甲乙方链路真实可感：续约、复盘、预算、试运行、数据归因都直接参与剧情推进。
- 男女主不是速配同盟，而是带旧上下级权力记忆的室友和项目对手。

## 主角与关系核心

- 姜栀：28 岁，澜曜互动资深客户经理，刚拿到代理客户组长头衔，能扛事、要体面，但最怕自己辛苦顶上的位置被一句“你只是运气好”收走。
- 裴述：31 岁，姜栀前上司，现任栖山咖啡品牌市场负责人，换岗后更在乎转化和利润，对旧关系有分寸，但不会为了旧情面牺牲业务判断。
- 关系基础：旧上下级权力记忆还在，生活上被同住捆住，工作上又一人拿方案、一人拿裁决权。

## 主线剧情

1. 第1-3章：姜栀升职接锅，裴述成了新室友兼甲方负责人。她在被误判和被压价中拿到一次试运行窗口，坐实继续带项目的资格。
2. 第4-6章：试运行数据起色，但旧投放链的内耗和公司内部抢功开始浮出。姜栀必须决定是保公司既有利益，还是保自己的职业判断。
3. 第7-9章：裴述的换岗理由、旧项目里被人为放大的预算问题曝光，两人从互相防备走到有限同盟，但代价是各自都要承受组织内压力。
4. 第10-12章：姜栀用项目结果和证据把代理组长坐实，裴述也完成品牌端内部整顿。两人的关系从“被迫同住的旧上下级”转成“知道边界也愿意并肩的人”。

## 第一卷规划

### 第一卷：升职接锅，同屋对冲

```yaml
卷目标: 让姜栀在前三章拿到代理位置后的第一口气，并把她和裴述的双重绑定锁死
起始章节: 1
结束章节: 6
核心冲突: 姜栀既要救回续约边缘的大客户，又要在和裴述的合租关系里守住职业边界
情绪走向: 接锅 -> 硬撑 -> 误判升级 -> 换账 -> 试探结盟 -> 代价外溢
```

## 前三章锁定

- 第001章：升职消息还没坐热，栖山咖啡先发来续约警报；姜栀回到新租房，发现新室友是前上司裴述，而他正是新甲方负责人。
- 第002章：同屋改稿和次日会面把关系从旧上下级推进到危险的“太熟”区，外界出现一次关系误判升级，直接反噬姜栀的职场声誉。
- 第003章：姜栀用一套更贴近转化的试运行方案拿下72小时窗口，职业线先小兑现，但她也因此更深地绑进裴述和公司内部的多方博弈。

## 主推进线与次级线

- 主线：姜栀保项目、坐稳职位。
- 次级线：姜栀和裴述在同住中重新校准边界与信任。
- 触动线：公司内部抢功、生活成本压力、同事流言、甲方团队的业绩压力。

## 结构锁定

- opening promise：第一章必须让读者同时看到职业困局和生活绑定。
- first-three payoff path：第1章挂钩，第2章误判升级，第3章小兑现。
- protagonist privilege boundary：姜栀赢项目只能靠更准的判断和更高的执行密度，不能靠裴述开后门。
- role tiers：
  - 核心角色：姜栀、裴述
  - 重要支撑：苏蔓、程竞、贺宁
  - 功能角色：同组同事、物业、司机、竞品代理
- relation tension：
  - 姜栀 vs 裴述：公私边界
  - 姜栀 vs 苏蔓：代理位置与资源分配
  - 裴述 vs 程竞：品牌面子与电商效率
- early gain residual risk：拿到试运行窗口后，姜栀还要面对公司内部抢功和室友关系越线的风险。

## 结局方向

- 事业结局：姜栀用结果把“代理”去掉，正式成为客户组长。
- 关系结局：两人不是因为合租才在一起，而是先证明彼此能在同一张现实账本上站住，再决定更进一步。

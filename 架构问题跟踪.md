## 状态怎么理解

- `open`：已经确认，必须排期处理
- `planned`：方向已明确，等进入执行
- `monitoring`：先不动，但必须盯着
- `deferred`：不是忘了，是当前有意延后
- `resolved`：已经正式收口

## 如果你担心自己忘记

每次碰到下面这些情况，先打开正式台账看一眼：

1. 你准备加新的 `state` 字段
2. 你准备新增 sidecar
3. 你准备改 `shared/`
4. 你准备调整 `novel-write / resume / workflow / maintenance`
5. 你准备接 MCP memory / 调度
6. 你准备从 `Genm` 真正同步 shared

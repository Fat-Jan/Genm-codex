## 状态怎么理解

- `open`：已经确认，必须排期处理
- `planned`：方向已明确，等进入执行
- `monitoring`：先不动，但必须盯着
- `deferred`：不是忘了，是当前有意延后
- `resolved`：已经正式收口

## 当前最值得优先记住的几条已收口边界

不是全部问题都要天天记，但下面这些最好一直有印象：

1. `shared/` 同步必须继续保持 governance-aware，不要绕过 allowlist 和 drift 护栏
2. `active-context` 只能做 projection / pointer，不能长成第二提示中心
3. MCP memory 只能做协调记忆，不能变成项目真值
4. 默认骨架仍是 `docs + skills + scripts + sidecars`，不要偷渡 monolithic runtime / plugin framework
5. 如继续增强 workflow / maintenance / state contract，要按台账中的触发条件复检，而不是默认它们永远安全

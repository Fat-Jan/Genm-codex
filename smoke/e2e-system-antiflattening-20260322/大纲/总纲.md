# 《系统把整栋楼都标成任务对象后，我先敲开了402的门》总纲

## 基础信息

```yaml
书名: 系统把整栋楼都标成任务对象后，我先敲开了402的门
题材: 系统流
目标平台: 番茄
预计章节数: 10
预计字数: 40000
```

## 一句话简介

失业后的陈曜搬进春来巷七码楼，当晚就绑定了一个把整栋楼住户都标成任务对象的系统。系统要他最快收齐改造签字、刷满邻里好感，可他越往里走越发现，这栋楼里没有一个人只是奖励包。

## 核心卖点

- 系统快节奏推进，但奖励逻辑和人情逻辑不断冲突
- 楼里住户人人都有自己的现实账，不是排队等主角拯救的 NPC
- 每一章都把“最快完成任务”和“把人当人看”往正面对撞

## 主角人设

- 姓名：陈曜
- 身份：刚被裁的游戏策划，现住春来巷七码楼
- 核心性格：会算、反应快、嘴不软，但还没坏到真把人命和积分等价
- 核心动机：先活下去，再弄清系统和七码楼改造到底是谁在借谁做局

## 主线剧情

1. 第1-3章：系统绑定、第一次任务、楼内规则与改造签字线浮出。
2. 第4-7章：陈曜在系统催逼和住户现实处境之间站队，安全改造签字战升级。
3. 第8-10章：系统真正目标和改造背后的利益链露面，陈曜必须决定到底是继续刷任务，还是拆系统的局。

## 第一卷规划

### 第一卷：七码楼不是任务大厅

```yaml
卷目标: 让主角意识到这栋楼里的人不是任务节点，而是各自拿命和钱在做选择的人
起始章节: 1
结束章节: 10
核心冲突: 系统要求最快收齐改造签字，现实里每个住户却都因为不同代价无法简单站队
情绪走向: 新奇 -> 便利 -> 起疑 -> 对撞 -> 反咬
```

## 关键关系

- 陈曜 vs 系统：效率与后果的冲突
- 陈曜 vs 方琴：从“任务对象”转成“必须尊重她的选择”
- 陈曜 vs 吴建国：从怀疑到有限同盟，但钱债裂口一直存在
- 陈曜 vs 何书宁：对立中的可撬动人，她保的是绩效和父亲手术费，不是物业脸面

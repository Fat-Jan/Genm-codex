# 使用流程总览

这份文档放在仓库主目录，目标只有一个：
让你不翻完整个 `docs/`，也能知道这个项目该怎么用。

更细的规则、阶段结论和专项说明仍然在 `docs/` 下，这里只保留主流程和必要跳转。

## 先做什么

第一次进入仓库，先做 3 步：

```bash
bash scripts/install-skills.sh
bash scripts/sync-shared-from-genm.sh --report
bash scripts/validate-migration.sh
```

说明：

- `install-skills.sh`：安装本地 skill 链接
- `sync-shared-from-genm.sh --report`：先只读检查共享资产状态；确认需要同步后，再执行真正的 `sync`
- `validate-migration.sh`：检查仓库结构和迁移状态是否正常

如果你就是想先看文档，不急着执行脚本，继续往下看即可。

## 先判断你现在处于哪种情况

### 1. 还没有项目

走这条主线：

`novel-init -> novel-genre -> novel-outline -> novel-package`

然后进入正文生产：

`novel-write -> novel-review -> novel-fix / novel-polish / novel-rewrite`

如果准备投稿：

`novel-precheck -> novel-package -> novel-export`

### 2. 已经有项目，想先看状态

优先用：

- `novel-status`
- `novel-query`
- `novel-index`

最小提示词：

```text
请使用 novel-status skill，给我一个 full 模式的项目状态面板。
```

### 3. 正在写作中

默认正文链路：

`novel-write -> novel-review -> novel-fix / novel-polish / novel-rewrite`

硬约束：

- 一次性最多只生成 `3` 章
- 超过 `3` 章要拆批
- 每批后都要过一次 `review` 或质量门

### 4. 设定、正文、state 开始脱节

优先用：

- `novel-sync`
- `novel-character`
- `novel-setting`

如果项目已经很长，还可以做 `thin-state`。

## 默认主流程

### 起盘

适合从零开始：

1. `novel-init`
2. `novel-genre`
3. 古代家族权力题材先补家谱/宅门真值表
4. 涉及朝堂、东宫、官场时再补官职真值表/权力层级图
5. `novel-outline`
6. `novel-package`

### 正文生产

适合稳定推进章节：

1. `novel-write`
2. `novel-review`
3. 根据问题类型进入：
   - `novel-fix`
   - `novel-polish`
   - `novel-rewrite`
4. 每 `3-5` 章或一个阶段后运行：
   - `novel-sync`

如果你想把维护动作挂成例行链路，可以用：

```bash
python3 scripts/post-task-maintenance.py <project_root> --trigger write
python3 scripts/project-maintenance.py <project_root>
```

### 投稿准备

1. `novel-precheck`
2. `novel-package`
3. `novel-export`

判断原则：

- `precheck = ready-now`：可以进入导出
- `precheck = revise-then-submit`：先回正文修
- `packaging-needs-update = yes`：先回包装层

### 持续学习

1. `novel-learn`
2. `novel-status`

适合已经积累了一批正文后，沉淀风格和稳定偏好。

## 常用辅助能力

不属于主线，但很常用：

- 查状态：`novel-status` / `novel-query` / `novel-index`
- 快速引用：`novel-retrieve`
- 设定维护：`novel-character` / `novel-setting` / `novel-foreshadowing` / `novel-sync`
- 回溯恢复：`novel-snapshot` / `novel-resume` / `novel-workflow`
- 环境检查：`novel-config` / `novel-test`

当前默认不纳入主线：

- `novel-scan`

## 如果你特别关心人物、关系和群像

优先把这几个 skill 连起来用：

- `novel-outline`
- `novel-write`
- `novel-review`
- `novel-precheck`

再配合阅读：

- [docs/anti-flattening-framework/README.md](docs/anti-flattening-framework/README.md)

这套规则主要用于限制：

- 主角特权过强
- 配角只剩功能位
- 阵营只有一个声道
- 推进过顺，没有真实代价和残账

## 最短判断

不知道从哪开始时，直接套这个判断：

1. 当前目录没有 `.mighty/state.json`
   - 从 `novel-init` 开始
2. 已经有 `.mighty/state.json`
   - 先跑 `novel-status`
3. 想继续写
   - `novel-write -> novel-review -> novel-fix / novel-polish / novel-rewrite`
4. 想投稿
   - `novel-precheck -> novel-package -> novel-export`
5. 想补维护
   - `novel-sync` 或维护脚本

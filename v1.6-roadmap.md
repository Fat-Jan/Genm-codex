# Genm-codex `v1.6` Roadmap

> Status: `active-mainline-roadmap`
>
> 这是当前主线 roadmap。`v1.5` 作为稳定基线与直接上游参考继续保留在根目录。

## 目的

`v1.6` 不再把“跨宿主支持”理解成一组零散安装补丁，而是进入：

1. 宿主能力事实真值层
2. 宿主安装 / 投影 / doctor 的薄消费层
3. 宿主支持状态与退化策略的显式边界
4. 主线 roadmap / README / `.ops` / 索引的一致化

重点不是：

- 继续先改 `install-skills.sh` 再补文档
- 先给每个宿主做一份平台特化说明
- 在没有证据和 contract 的前提下扩大“已支持”承诺

而是：

- 先把宿主事实收成 repo-owned truth
- 再让脚本、文档、doctor 与 support table 从真值层派生
- 用 phase / status / verify 把这条线正式纳入项目主线，而不是继续停留在单页说明或工作区实现

- **跨宿主基础层 contract 化**

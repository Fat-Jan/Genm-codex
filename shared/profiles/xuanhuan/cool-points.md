```markdown
加权爽点密度 = Σ(爽点权重) / (字数/1000)
```

## 玄幻题材爽点类型

### 1. 越级反杀（权重 1.5）

**定义**：低境界击败高境界，展示主角特殊手段

**触发词**：越级、反杀、不可能、不可思议、碾压、蝼蚁

**设计要点**：
- 必须有合理理由（金手指、法宝、功法压制）
- 必须有代价或限制
- 必须有铺垫（前文暗示主角的特殊能力）

**示例**：

```markdown
# 越级反杀爽点示例

【铺垫】前文主角获得神秘传承，可以短暂爆发元婴级别的攻击
【代价】使用后虚弱三天，消耗十年寿元

【正文】
"区区金丹，也敢挡本座的路？" 元婴修士满脸轻蔑。

林轩没有说话，只是眼中闪过一丝决然。

下一瞬，一道青芒从他眉心射出——那是不可能属于金丹修士的力量！

元婴修士脸色大变："这是...元婴级别的攻击？！"

青芒穿透了他的护体灵光，在他肩膀上留下一个血洞。

"怎么可能！你明明只是金丹期！"

林轩缓缓开口，声音沙哑："我说过，不要轻视任何人。"

话音落下，他的身体已经摇摇欲坠。这一击，几乎抽干了他的全部灵力和寿元。

但他赢了。
```

### 2. 机缘获得（权重 1.4）

**定义**：获得传承、宝物、功法

**触发词**：传承、宝物、机缘、造化、觉醒、获得

**设计要点**：
- 机缘来源要有合理性
- 机缘与主角要有契合度
- 机缘要有成长空间

**示例**：

```markdown
# 机缘获得爽点示例

林轩小心翼翼地探出一缕神识，心神巨震。

这古卷中，竟记载着一门上古剑诀！

"青云剑诀...天阶上品..." 林轩喃喃自语，手指微微颤抖。

这可是连青云宗都没有的天阶功法！

突然，一道虚影从古卷中浮现，目光如炬：

"吾乃青云子，今传汝青云剑诀。但有一条件——替吾了却一桩心愿。"

林轩没有犹豫，双膝跪地："弟子愿意。"
```

### 3. 境界突破（权重 1.3）

**定义**：突破境界，实力飞跃

**触发词**：突破、境界、筑基、金丹、元婴

**设计要点**：
- 必须有铺垫和积累
- 突破过程要有波折
- 突破后要有展示

### 4. 打脸爽（权重 1.2）

**定义**：被轻视后展现实力

**触发词**：轻视、嘲讽、看不起、废物、震惊、脸色大变

**设计要点**：
- 需要先建立信息差
- 打脸要有层次感
- 反应要有差异性

**打脸模板**：

```markdown
## 打脸场景设计模板

### 第一步：铺垫信息差
- 主角有隐藏实力/身份
- 对方不知情，表现出轻视

### 第二步：建立反差
- 对方表现傲慢
- 周围人反应（嘲讽/看戏/担忧）

### 第三步：打脸时刻
- 主角展现实力
- 对方震惊/不敢置信

### 第四步：连锁反应
- 对方脸色变化
- 周围人反应反转
- 后续影响
```

### 5. 逆袭爽（权重 1.1）

**定义**：从劣势到优势的逆转

**触发词**：逆转、翻盘、反败为胜、扭转、绝地

**设计要点**：
- 先处于劣势
- 有转折契机
- 逆转要合理

## 爽点组合技巧

### 组合加成规则

| 组合方式 | 加成值 | 说明 |
|----------|--------|------|
| 连续爽点 | +0.2 | 爽点连续出现 |
| 铺垫后爆发 | +0.3 | 铺垫超过500字后爆发 |
| 对比反差 | +0.2 | 前后状态形成强烈对比 |
| 多重叠加 | +0.1/层 | 多个爽点同时出现，上限+0.3 |

### 爽点分布

```yaml
# 爽点间隔建议

小爽点:
  间隔: 800-1200字
  类型: 小打脸、小收获、小突破

中爽点:
  间隔: 2000-3000字
  类型: 中等打脸、重要收获、境界小进

大爽点:
  位置: 章末或卷末
  类型: 重大打脸、大境界突破、核心机缘
```

## 爽点禁忌

### 绝对禁止

| 禁忌 | 说明 | 后果 |
|------|------|------|
| 无铺垫打脸 | 没有建立信息差就打脸 | 读者无感 |
| 无代价越级 | 越级战斗没有代价 | 破坏力量体系 |
| 爽点过密 | 连续爽点无喘息 | 审美疲劳 |
| 同质化爽点 | 同类型爽点重复出现 | 降低效果 |

### 需要预警

| 情况 | 说明 | 建议 |
|------|------|------|
| 爽点间隔过长 | 超过1500字无爽点 | 插入小爽点 |
| 爽点密度过高 | 连续多个大爽点 | 添加过渡段落 |
| 爽点类型单一 | 只用一种爽点 | 丰富爽点类型 |

```markdown

[ ] 越级战斗有代价或特殊手段
[ ] 打脸场景有信息差建立
[ ] 机缘获得有合理性
[ ] 爽点类型多样化
[ ] 大爽点位置合理（章末/卷末）
```

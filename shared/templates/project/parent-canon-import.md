# Parent Canon 导入指南

> 从原作书籍导入设定，确保番外与原作的一致性
>
> 使用场景：撰写番外、外传、衍生作品时

## Step 1: 确定原作书籍

### 检测可用书籍

```bash
# 扫描 .mighty/books/ 目录
可用书籍列表:
  - book1: 《书名1》 (genre: 修仙, chapters: 100)
  - book2: 《书名2》 (genre: 玄幻, chapters: 50)
  - ...
```

### 用户选择

```yaml
如果未指定 parent:
  显示书籍列表供选择

如果指定的 parent 不存在:
  提示错误并显示可用列表

如果存在多个相关书籍:
  询问是否需要导入多本 Canon
```

## Step 2: 加载核心 Canon

### 2.1 项目元信息 (必须)

**来源**: `.mighty/books/{parent}/state.json`

```yaml
提取字段:
  meta:
    - title: 原作书名
    - genre: 题材类型
    - platform: 目标平台
    - target_words: 目标字数

  entities.characters.protagonist:
    - name: 主角姓名
    - aliases: 别名/称号
    - power: 力量体系状态
    - location: 当前位置
    - inventory: 持有物品
    - abilities: 能力列表
    - status: 状态效果
    - current_goals: 当前目标
```

### 2.2 主角详细设定 (必须)

**来源**: `.mighty/books/{parent}/设定集/角色/主角.md`

```markdown
提取内容:
  - 基础信息（姓名、年龄、外貌）
  - 性格特征（核心价值观、行为模式）
  - 成长经历（关键事件）
  - 能力设定（技能、天赋）
  - 人际关系（重要关系人）
  - 标志性特点（口头禅、习惯动作）
```

### 2.3 力量体系设定 (必须)

**来源**: `.mighty/books/{parent}/设定集/力量体系.md`

```markdown
提取内容:
  - 体系名称与层级
  - 升级规则与限制
  - 能力类型与效果
  - 特殊机制（瓶颈、天劫等）
  - 与其他体系的关系
```

### 2.4 共享宇宙设定 (如果有)

**来源**: `.mighty/universe/world-settings.md`

```markdown
提取内容:
  - 统一的世界观规则
  - 跨书共享设定
  - 正史时间线定位
```

## Step 3: 选择性加载

### 根据番外类型加载

#### 前传番外

```yaml
额外加载:
  - 主角背景故事（state.json#protagonist.backstory）
  - 提及的历史事件（大纲/总纲.md中的背景设定）
  - 势力的早期状态

注意:
  - 不得透露主线未揭示的信息
  - 埋下与主线呼应的伏笔
```

#### 后传番外

```yaml
额外加载:
  - 大结局章节（确认结局状态）
  - 未解决的支线线索
  - 主角的最终状态

注意:
  - 必须与结局一致
  - 可以适度开放式结局
```

#### 外传/角色篇

```yaml
额外加载:
  - 聚焦角色的详细设定
  - 该角色在主线中的关键出场章节
  - 该角色的关系网

注意:
  - 保持与主线视角的一致性
  - 不得与主线剧情冲突
```

#### IF线/平行宇宙

```yaml
额外加载:
  - 分叉点之前的所有设定

注意:
  - 分叉点之后可以自由发挥
  - 必须明确标注为非正史
```

### 根据时间定位加载

```yaml
如果指定了 time_point:
  加载该时间点前的状态:
    - state.json 中对应 chapter 的快照
    - 该章节附近的内容摘要
    - 角色的当前状态
```

## Step 4: 一致性校验

### 4.1 角色设定校验

```markdown
**性格一致性检查**:
- [ ] 对话风格是否符合原作
- [ ] 决策模式是否与性格匹配
- [ ] 面对压力时的反应是否一致
- [ ] 核心价值观是否保持

**能力状态检查**:
- [ ] 该时间点应掌握的能力
- [ ] 该时间点不应掌握的能力
- [ ] 境界/等级是否符合时间线
- [ ] 持有物品是否正确
```

### 4.2 时间线校验

```markdown
**前置条件检查**:
- [ ] 事件A发生时，事件B是否已发生
- [ ] 角色C此时是否认识角色D
- [ ] 物品E此时是否已被获取
- [ ] 势力F此时的状态是否正确

**因果关系检查**:
- [ ] 番外事件不会导致时间悖论
- [ ] 番外设定不会与主线矛盾
- [ ] 番外结局不会剧透主线
```

### 4.3 世界观校验

```markdown
**设定一致性检查**:
- [ ] 力量体系规则是否统一
- [ ] 势力关系是否匹配
- [ ] 地理设定是否一致
- [ ] 特殊规则是否遵守
```

### 4.4 校验报告

```yaml
输出格式:
  校验项目数: {total_checks}
  通过项目数: {passed_checks}
  警告项目数: {warning_count}
  冲突项目数: {conflict_count}

  冲突详情:
    - 项目: "{描述}"
      原作设定: "{原值}"
      当前设定: "{新值}"
      建议: "{解决方案}"
```

## Step 5: 生成 Canon 摘要

### 摘要模板

```markdown
## Parent Canon 摘要

### 原作信息
- **书名**: {title}
- **题材**: {genre}
- **平台**: {platform}
- **总章节**: {total_chapters}

### 主角状态 ({time_point})
- **姓名**: {protagonist.name}
- **身份**: {protagonist.aliases}
- **境界**: {protagonist.power}
- **位置**: {protagonist.location}
- **目标**: {protagonist.current_goals}

### 关键设定
{power_system_summary}

### 相关角色
{character_list}

### 写作约束
**必须遵守**:
- 约束1
- 约束2

**建议遵循**:
- 建议1
- 建议2

**注意事项**:
- 注意1
- 注意2
```

### Canon 引用标记

```markdown
在写作过程中，需要标记引用的Canon来源:

[Canon: state.json#entities.characters.protagonist.name]
[Canon: 力量体系.md#境界划分]
[Canon: 第50章#角色状态]
```

## 导入文件清单

### 必须文件

| 文件路径 | 用途 | 优先级 |
|----------|------|--------|
| `.mighty/books/{parent}/state.json` | 项目元信息、主角状态 | P0 |
| `.mighty/books/{parent}/设定集/角色/主角.md` | 主角详细设定 | P0 |
| `.mighty/books/{parent}/设定集/力量体系.md` | 力量体系规则 | P0 |

### 建议文件

| 文件路径 | 用途 | 优先级 |
|----------|------|--------|
| `.mighty/books/{parent}/大纲/总纲.md` | 了解主线框架 | P1 |
| `.mighty/books/{parent}/设定集/角色/{focus}.md` | 聚焦角色设定 | P1 |
| `.mighty/universe/world-settings.md` | 共享世界观 | P1 |

### 按需文件

| 文件路径 | 用途 | 优先级 |
|----------|------|--------|
| `chapters/第{N}章.md` | 时间线定位 | P2 |
| `设定集/势力/*.md` | 涉及势力时 | P2 |
| `设定集/地点/*.md` | 涉及地点时 | P2 |

## 常见问题处理

### Q1: 原作设定不完整怎么办？

```yaml
处理方案:
  如果关键设定缺失:
    1. 从已完成的章节中推断
    2. 使用合理的默认值
    3. 在Canon摘要中标记为"推断值"
    4. 建议用户后续补充原作设定
```

### Q2: 番外需要修改原作设定怎么办？

```yaml
处理方案:
  评估修改类型:
    扩展性修改: 可以添加原作未提及的细节
    兼容性修改: 可以在不矛盾的前提下微调
    冲突性修改: 需要明确标注为IF线/平行宇宙
```

### Q3: 多本原作的Canon如何合并？

```yaml
处理方案:
  检查Canon冲突:
    无冲突: 直接合并
    有冲突:
      - 标记冲突点
      - 让用户决定以哪本为准
      - 记录合并规则
```

### Q4: Canon导入后原作更新了怎么办？

```yaml
处理方案:
  版本检查:
    1. 记录Canon导入时的版本
    2. 定期提示检查原作更新
    3. 如有重大更新，建议重新导入
```

## 自动化导入命令

### 完整导入

```bash
# 导入指定书籍的全部Canon
/novel-spinoff parent=book1 type=前传 --import-canon=full
```

### 选择性导入

```bash
# 仅导入角色设定
/novel-spinoff parent=book1 --import-canon=characters

# 仅导入时间线定位
/novel-spinoff parent=book1 time_point="第50章" --import-canon=timeline

# 仅导入特定角色
/novel-spinoff parent=book1 focus_character=林天 --import-canon=focused
```

## 校验清单

在提交番外前，使用以下清单自查：

```markdown
## Canon 一致性自查清单

### 角色一致性
- [ ] 主角性格符合原作
- [ ] 对话风格与原作一致
- [ ] 能力状态符合时间线
- [ ] 人际关系正确

### 时间线一致性
- [ ] 事件顺序合理
- [ ] 前置条件满足
- [ ] 无时间悖论
- [ ] 与主线不冲突

### 世界观一致性
- [ ] 力量体系规则统一
- [ ] 势力关系匹配
- [ ] 地理设定一致
- [ ] 特殊设定遵守

### 番外类型合规
- [ ] 前传：未剧透主线
- [ ] 后传：与结局一致
- [ ] 外传：视角合理
- [ ] IF线：明确标注
```

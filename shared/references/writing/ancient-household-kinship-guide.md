# 古代宅门亲疏谱系校验指南

## 目的

这份指南只做一件事：

- 防止 `宫斗宅斗 / 古代言情 / 家族权力` 项目里出现“关系词好看，但谱系不闭合”的硬伤。

它不是全套历史制度教材，而是生成链上游必须过的最小事实门。

---

## 最小真值表

在生成书名、总纲、包装、角色卡、正文之前，先补齐这张最小表：

- `father`
- `wives`
  - 原配 / 继室 / 妾室 / 通房
- `children`
  - 姓名
  - 性别
  - 生母
  - 嫡 / 庶
  - 齿序
  - 当前状态
    - 在府
    - 出嫁
    - 早夭
    - 过继
    - 长居外地

如果这张表没闭合，不要直接进入包装层，更不要把关系词写进书名。

---

## 硬规则

### 1. 关系词必须能回到生母与法统

- `嫡女 / 嫡子` 必须对应“法定正室所出”
- `庶女 / 庶子` 必须对应“非正室所出”
- `继母` 只说明“不是生母”，不自动等于“她的孩子也是庶出”

默认处理原则：

- 如果一个女人在项目里承担的是后宅主母 / 继室位置，她所生子女默认优先校验为 `嫡出`，除非项目明确写了别的法统安排。

### 2. 齿序称谓必须闭合

- 用了 `二姑娘 / 三姑娘 / 二少爷`，就必须交代前序子嗣去向。
- 可以不详写剧情，但至少要在设定层交代：
  - 已出嫁
  - 早夭
  - 不在府中
  - 同父异母且另居

补充提醒：

- `大姑娘 / 二姑娘` 这类称谓是故事内的当前序位识别，不是可以随手拿来增强点击的空标签。
- 只要把它写进书名、简介、总纲，就必须先把同父同代子女序列补齐。

### 3. 书名关系词必须和正文真值一致

这些词不能只因为“点得动”就硬塞：

- 庶妹
- 嫡姐
- 嫡母
- 继妹
- 二姑娘
- 长房 / 二房 / 三房

只要书名、简介、总纲、角色卡、正文任意一层不一致，都视为同一个根因问题，不是文案问题。

### 4. `嫡庶压制` 不是装饰标签

当项目打上：

- `嫡庶压制`
- `继母陷害`
- `后宅翻盘`

就必须明确：

- 谁是法统上的嫡出
- 谁在借嫡位压人
- 主角为什么处于低位
- 对手的身份优势从哪来

### 5. 当前项目里最常见的错误组合

以下组合默认判为高风险：

- `庶女主角 + 庶妹对手 + 嫡庶压制`
  - 但没有正妻所出女儿去向
- `继母主压制 + 其亲生女儿写成庶妹`
  - 但没有额外交代法统例外
- `二姑娘 / 三姑娘`
  - 但设定层没有前序子女

### 6. 排辈 / 字辈不要机械套用

- 家谱里的 `字辈 / 行辈 / 派语` 更常用于宗族世系识别，不等于所有古代女性角色都会稳定按同一套辈字起名。
- 女频古代小说里，闺阁女子更常见的是：
  - 姓 + 名
  - 排行称呼
  - 某氏
  - 某姑娘
  - 某娘子
- 如果项目想写“同辈共用某个字”，必须先确定这是该家族的明确设定，不要因为现代读者熟悉“辈分字”就把所有男女角色都做成同模板命名。

默认建议：

- 排辈用于家族谱系表时可以强约束
- 日常人物命名时不要默认强绑定到同一辈字
- 女性角色尤其不要机械同字堆叠，否则很容易既失真又撞名

## 信源使用顺序

古代题材不要只靠“我感觉古人应该这样”。

建议按这个顺序做：

1. **项目内真值**
   - `state`
   - `总纲`
   - `角色卡`
   - 现有正文
2. **概览型资料**
   - `Wikipedia / Wikiwand / 百科`
   - 作用：快速确认术语范围和常见制度轮廓
3. **确认型资料**
   - 学术论文
   - 研究机构文章
   - 政府 / 高校 / 图书馆谱牒资料
4. **最后决策**
   - 如果资料之间仍有时期差异或礼制差异，不要装作只有一种绝对答案
   - 直接把故事所在朝代、门第层级和本家规矩写清

原则：

- wiki 类资料适合先定问题边界
- 学术 / 谱牒 / 研究机构资料适合锁最终口径
- 两层都没过，不要把关系词写死进包装主方案

## 推荐参考方向

### 概览层

- `嫡庶`
- `庶出子女`
- `女性人名`
- `表字`

这些条目适合先判断：

- 嫡庶的基本含义
- 继承顺序和法统观念
- 女子名字、排行字、称谓的常见写法
- 哪些词只是礼制概念，哪些词能直接落到人物称呼

### 确认层

- 家谱 / 宗谱中的妻妾书写规则
- 继娶、承祧、过继、宗祧
- 亲属称谓的泛化与限度
- 明清以后宗族书写对嫡、继、妾、子女归母的处理

这层更适合解决：

- “继母的亲生女儿能不能直接写成庶妹”
- “二姑娘前面的大姑娘必须如何交代”
- “这个家族该不该设辈字”
- “女子角色名该按谱字走，还是按称谓走”

## 生成前检查清单

- [ ] 父系与妻妾结构已列清
- [ ] 每个关键子女的生母已明确
- [ ] 嫡庶身份能回到法统，不靠想当然
- [ ] 齿序称谓有前序解释
- [ ] 书名关系词与总纲、角色卡、正文一致
- [ ] 若关系词只服务点击，不服务设定，先删关系词再改包装

---

## 一票否决项

出现以下任一项，包装和投稿直接降级：

- `嫡庶压制` 但看不出谁是嫡谁是庶
- `庶妹` 是核心卖点，但谱系里不存在稳定解释
- `二姑娘` 是关键称谓，但没有大姑娘去向
- 角色卡、总纲、正文对同一角色的嫡庶定义不同

建议状态：

- `packaging-needs-update: yes`
- `submission-status: do-not-submit`

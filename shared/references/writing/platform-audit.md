# 平台审核规避指南

> 针对番茄、七猫、起点、晋江四大平台的 AI 检测与内容审核策略
> 版本：v1.0

---

## 一、朱雀检测原理

### 1.1 四大检测维度

| 维度 | 检测内容 | 触发信号 | 风险等级 |
|------|----------|----------|----------|
| **词汇指纹** | AI 高频词、固定搭配 | "淡淡的"、"微微"、"不禁" | 高 |
| **句式模式** | 机械重复结构 | 三段式列举、二元排比 | 高 |
| **语义连贯性** | 上下文衔接痕迹 | 过度解释、填充短语 | 中 |
| **信息密度** | 单位字数信息量 | 废话过多、绕圈表达 | 中 |

### 1.2 检测机制说明

**词汇指纹识别**
- 建立 AI 写作语料库作为对比基准
- 统计特定词汇的出现频率和共现模式
- 标记偏离人类自然表达的用词习惯

**句式模式匹配**
- 识别固定句式模板（如"不仅...而且...更..."）
- 检测段落长度均匀度
- 分析句子结构重复率

**语义连贯性分析**
- 检查上下文的逻辑衔接是否过于"完美"
- 识别人类写作中不应存在的过度解释
- 标记缺乏个人视角的中立叙述

**信息密度评估**
- 计算有效信息量与总字数比
- 识别冗余修饰和填充内容
- 标记绕圈表达方式

### 1.3 平台检测严格度排名

```markdown
最严格 → 番茄小说 ≈ 晋江文学城 → 七猫小说 → 起点中文网 → 最宽松
```

**番茄小说**
- AI 检测最为敏感
- 签约前全文扫描
- 疑似 AI 内容要求人工审核

**晋江文学城**
- 人工+机器双重审核
- 对文笔和原创性要求高
- 文风同质化会被标记

**七猫小说**
- 侧重爽点密度检测
- AI 痕迹容忍度中等
- 更关注内容是否吸引下沉市场

**起点中文网**
- 相对宽松
- 更重视创意和世界观
- AI 辅助写作不禁止，但纯 AI 内容会被拒

## 二、平台审核红线

### 2.1 绝对禁区（全平台）

| 类别 | 具体内容 | 后果 |
|------|----------|------|
| **政治敏感** | 现行政治体制、领导人、敏感事件 | 永久封号 |
| **色情低俗** | 直接性行为描写、性暗示过度 | 下架+封号 |
| **极端暴力** | 详细杀人过程、血腥描写 | 下架 |
| **迷信邪教** | 宣扬封建迷信、邪教组织 | 永久封号 |
| **未成年人** | 未成年人性暗示、不良诱导 | 下架+法律责任 |

### 2.2 各平台特殊禁忌

#### 番茄小说

| 禁忌类型 | 具体表现 | 替代方案 |
|----------|----------|----------|
| 敏感词 | "杀"、"死"、"血"（高频） | 用"解决"、"终结"、"猩红"替代 |
| 现实影射 | 影射真实人物、事件 | 完全虚构化 |
| 价值观问题 | 拜金、三观不正主角 | 给主角正向动机 |
| 抄袭嫌疑 | 与热门作品高度相似 | 核心设定差异化 |

#### 七猫小说

| 禁忌类型 | 具体表现 | 替代方案 |
|----------|----------|----------|
| 节奏过慢 | 5万字无爽点 | 每3000字一个钩子 |
| 过于文艺 | 描写过多影响节奏 | 减少环境描写，增加对话 |
| 主角弱势 | 长期被虐无反击 | 虐后有快速翻盘 |

#### 起点中文网

| 禁忌类型 | 具体表现 | 替代方案 |
|----------|----------|----------|
| 设定抄袭 | 与热门作品雷同 | 核心机制创新 |
| 世界观漏洞 | 逻辑自相矛盾 | 建立设定文档，保持一致 |
| 剧情老套 | 完全按套路走 | 加入反转和意外 |

#### 晋江文学城

| 禁忌类型 | 具体表现 | 替代方案 |
|----------|----------|----------|
| 文笔稚嫩 | 对话生硬、描写单一 | 加强文笔训练 |
| 情感虚假 | 感情线突兀 | 铺垫足够心理活动 |
| 同质化 | 与他人作品雷同 | 找到独特切入点 |

### 2.3 敏感词替换词表

#### 暴力/死亡类

| 原词 | 番茄/七猫替代 | 起点/晋江替代 |
|------|---------------|---------------|
| 杀死 | 解决、终结、除掉 | 斩杀、击败 |
| 死亡 | 陨落、消逝、长眠 | 身亡、阵亡 |
| 血腥 | 猩红、暗红、铁锈味 | 血腥味 |
| 尸体 | 遗骸、躯壳 | 尸身 |
| 屠杀 | 清洗、肃清 | 屠戮（慎用） |

#### 色情/情感类

| 原词 | 替代写法 |
|------|----------|
| 直接性描写 | 转为意境描写、省略号留白 |
| 裸露 | 衣衫不整、肌肤外露 |
| 床戏 | 次日清晨、红烛熄灭 |
| 亲吻详细描写 | 一吻封缄、唇瓣相接 |

#### 政治/社会类

| 原词 | 替代写法 |
|------|----------|
| 现实国家名 | 完全虚构国名 |
| 现实地名 | 虚构地名或模糊化 |
| 政府机构 | 宗门、家族、组织 |
| 社会矛盾 | 转化为正邪对立 |

## 三、规避策略

### 3.1 词汇层面

#### 禁用词清单（必删）

**AI 高频词**
- 淡淡的、微微、不禁、竟然、竟、顿时
- 此外、至关重要、深入探讨、强调
- 格局、织锦、复杂性、关键性
- 标志着、见证了、体现了、作为

**网文禁用词**
- 嘴角微微上扬 → 嘴角上扬
- 眼神微微一动 → 眼神一动
- 身体微微颤抖 → 身体颤抖
- 淡淡的一笑 → 一笑

#### 词汇替换策略

```markdown
替换公式：抽象词 → 具体描写

❌ 他感到非常愤怒
✅ 他捏碎了手里的杯子

❌ 她看起来很伤心
✅ 她背过身去，肩膀抽动

❌ 场面一片混乱
✅ 椅子倒了，茶杯碎在地上，有人在尖叫
```

### 3.2 句式层面

#### 打破三段式法则

```markdown
❌ 他不仅武功高强，而且智谋过人，更是心怀天下
✅ 他武功高，脑子也够用。至于心怀天下？那是顺带的。

❌ 修炼有三个境界：筑基、金丹、元婴
✅ 修炼分筑基和金丹。过了金丹，有人能到元婴，有人一辈子卡在那。
```

#### 变化句子长度

```markdown
❌ [长句] 林天站在山巅，看着远处的云海，心中思绪万千，想起了这些年的经历。
[长句] 他从一个小山村走出，历经千辛万苦，终于达到了今天的成就。
[长句] 但这只是开始，前方还有更长的路要走，更多的挑战在等待着他。

✅ [短句] 林天站在山巅。云海翻涌。
[中句] 他想起了那个小山村。
[长句] 十五年了，从被人踩在脚下到如今这一步，靠的不是天赋，是命硬。
[短句] 前方还有路。他抬起脚。
```

#### 避免二元对比陷阱

```markdown
❌ 这不仅是一场战斗，更是一次考验
✅ 这一战，输了就死。

❌ 她不只是美丽，而且聪慧
✅ 她好看。脑子也好使。
```

### 3.3 结构层面

#### 打破完美节奏

```markdown
❌ 每章3000字，每章末尾必留钩子
✅ 有的章2000字结束，有的章5000字，钩子不一定在章末

❌ 每三章一个小高潮，每十章一个大高潮
✅ 高潮来临看剧情需要，可以提前或延后
```

#### 变化段落结构

```markdown
❌ [描写] [对话] [心理] [动作] - 固定顺序重复
✅ [对话] [动作] [对话] [描写] - 随机组合
```

### 3.4 内容层面

#### 展示代替说明

```markdown
❌ 说明：他是一个冷酷无情的人
✅ 展示：他看着那个人倒下，眼神没有波动，转身离开

❌ 说明：她很害怕
✅ 展示：她的手在抖，想说什么，嘴唇动了动，没发出声音
```

#### 删除过度解释

```markdown
❌ 他这样做是因为……（解释动机）
✅ 直接写他做了什么，动机通过行为暗示

❌ 这意味着……（解释意义）
✅ 让读者自己体会
```

#### 加入"不完美"细节

```markdown
❌ 完美的描写：阳光正好，微风不燥，气氛恰到好处
✅ 不完美的真实：阳光有点刺眼，风把纸吹跑了，气氛有点尴尬
```

### 3.5 平台针对性策略

#### 番茄小说

| 策略 | 具体做法 |
|------|----------|
| 降低 AI 词密度 | 每1000字检查，禁用词不超过2个 |
| 增加口语化 | 对话占比30%以上 |
| 短句优先 | 平均句长控制在25字以内 |
| 个人视角 | 第三人称中穿插主角内心吐槽 |

#### 七猫小说

| 策略 | 具体做法 |
|------|----------|
| 强化爽点 | 每2000字一个小反转 |
| 节奏紧凑 | 减少环境描写，加快剧情 |
| 直白表达 | 不绕弯，直接给结果 |
| 情绪外放 | 情绪通过对话和动作表达 |

#### 起点中文网

| 策略 | 具体做法 |
|------|----------|
| 设定创新 | 核心机制要有新意 |
| 世界构建 | 详细但不啰嗦的设定展现 |
| 剧情反转 | 不按套路出牌 |
| 人物立体 | 配角也要有独立故事线 |

#### 晋江文学城

| 策略 | 具体做法 |
|------|----------|
| 文笔打磨 | 注重描写的美感 |
| 情感细腻 | 心理活动丰富但自然 |
| 人物真实 | 有缺点的主角更受欢迎 |
| 视角独特 | 找到不一样的讲故事角度 |

## 四、写作后自检清单

### 4.1 AI 痕迹检查

#### 词汇检查

- [ ] 全文搜索"淡淡的"，确保0处
- [ ] 全文搜索"微微"，确保0处
- [ ] 全文搜索"不禁"，确保0处
- [ ] 全文搜索"竟然"，替换为"居然"或直接删除
- [ ] 全文搜索"顿时"，替换为"立刻"或直接删除
- [ ] 检查是否存在"标志着/见证了/体现了"
- [ ] 检查是否存在"不仅...而且...更"
- [ ] 检查是否存在"此外/至关重要/深入探讨"

#### 句式检查

- [ ] 段落长度是否过于均匀（每段200-300字为佳，要有长短变化）
- [ ] 是否存在连续三段以上结构相似的段落
- [ ] 每章开头是否都是"XX看着/想着"
- [ ] 句子长度是否有变化（长短交错）
- [ ] 是否存在明显的三段式列举

#### 内容检查

- [ ] 是否存在过度解释动机
- [ ] 是否存在大量"说明"而非"展示"
- [ ] 是否存在完美的场景描写
- [ ] 是否有个人视角和观点
- [ ] 是否承认复杂性和混乱

### 4.2 平台特定检查

#### 番茄小说

- [ ] 敏感词已替换（杀→解决/终结，死→陨落/消逝）
- [ ] 现实地名已虚构化
- [ ] AI 词密度每千字不超过2个
- [ ] 口语化对话占比30%以上

#### 七猫小说

- [ ] 每3000字有爽点或钩子
- [ ] 节奏是否够快（检查是否有拖沓段落）
- [ ] 主角是否长期处于弱势
- [ ] 环境描写是否过多

#### 起点中文网

- [ ] 核心设定是否有创新点
- [ ] 世界观是否存在逻辑漏洞
- [ ] 剧情是否过于套路化
- [ ] 配角是否有独立故事线

#### 晋江文学城

- [ ] 文笔是否过于稚嫩
- [ ] 情感线是否铺垫足够
- [ ] 是否与热门作品过于相似
- [ ] 人物是否立体真实

### 4.3 通用质量检查

- [ ] 错别字检查
- [ ] 标点符号使用规范
- [ ] 人物名称保持一致
- [ ] 时间线逻辑通顺
- [ ] 章节字数符合平台要求

## 五、快速应对指南

### 5.1 被标记 AI 嫌疑怎么办

**第一步：定位问题**
1. 使用禁用词清单全文搜索
2. 检查段落长度均匀度
3. 审查是否有过度解释

**第二步：针对性修改**
1. 删除所有 AI 高频词
2. 打破均匀段落，加入短句段落
3. 展示代替说明

**第三步：人工润色**
1. 增加个人视角和吐槽
2. 加入不完美的真实细节
3. 口语化处理部分叙述

### 5.2 紧急情况快速脱 AI 味

```markdown
10分钟急救方案：

1. 搜索替换（3分钟）
   - 淡淡的 → [删除]
   - 微微 → [删除]
   - 不禁 → [删除]
   - 竟然 → 居然
   - 顿时 → 立刻

2. 段落调整（4分钟）
   - 找到最长的段落，拆分成两段
   - 在最长的段落后面加一个短句段落（5-10字）

3. 快速润色（3分钟）
   - 在任意位置加入一句主角内心吐槽
   - 把一个"说明"改成"展示"
```

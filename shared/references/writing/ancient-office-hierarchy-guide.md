# 古代官职层级校验指南

## 目的

这份指南用于拦截另一类高频古代题材幻觉：

- 官职名对了，但权力链错了
- 品级写对了，但实际能管谁写错了
- 内廷、外朝、东宫、地方、军中混成一锅
- 明清官名和唐宋官制混搭
- 爵位、散官、实职、差遣、诰命被当成同一种东西

它不追求写成完整官制史，而是给生成链一个最小可执行门槛。

---

## 最小原则

### 1. 先锁时代口径，再锁官名

古代项目必须先回答：

- 是明确朝代
- 还是架空，但参考某朝制度
- 还是高度混融架空

如果这一步不写清，后面很容易出现：

- `巡抚` 和 `三省六部`
- `节度使` 和 `军机处`
- `内阁首辅` 和 `中书令`

这种跨代混用。

默认要求：

- 只要核心冲突涉及朝堂、宫廷、地方治理、军政、宗室封爵，就先写 `制度参考口径`。

### 2. 官名不等于权力

至少要分清：

- `实职`
- `品秩`
- `散官`
- `差遣 / 使职`
- `爵位`
- `勋位`
- `命妇 / 封号`
- `宫职 / 东宫官`

常见误区：

- 把品级高低直接等于实际权力大小
- 把爵位当行政职务
- 把诰命夫人当作朝廷实权职位
- 把东宫属官当成外朝部长

### 3. 权力链要落到“谁能命谁”

只写“他是正五品”不够。

还要回答：

- 他归谁节制
- 他能调谁
- 他只能上奏，还是能直接下令
- 他的权力来自职官本身，还是来自差遣、兼衔、皇帝宠信、家族背景

### 4. 官场戏优先校验五条线

- 皇帝 / 太后 / 皇后 / 太子线
- 外朝中枢线
- 地方行政线
- 军政边镇线
- 宗室 / 勋贵 / 命妇线

只要主角或核心对手站在这些线里，真值表就要先补。

---

## 最常见错误组合

- `尚书` 被写成六部里某司的普通办事官
- `侍郎` 被写成纯副职，剧情里却拥有独断一部的实权
- `巡抚` 被写成固定常设省级长官，同时又套进前代中央结构
- `知府 / 知州 / 知县` 的上下级链错位
- `国公 / 侯爷 / 伯府` 被写成行政官署
- `诰命夫人` 被写成可直接调官断案
- `太子属官` 被写成可以直接压六部
- `正五品` 被机械理解为“所有朝代都差不多”

---

## 生成前检查清单

- [ ] 已锁 `明确朝代` 或 `参考制度口径`
- [ ] 已区分实职、品秩、爵位、命妇、宫职
- [ ] 核心角色的官名能回到具体权力链
- [ ] 主要官职之间的上级、平级、下级关系已写明
- [ ] 地方官和京官没有混成一层
- [ ] 书名、简介、总纲里若写官名，已确认不是空喊职位

---

## 一票否决项

出现以下任一项，包装和投稿应降级：

- 朝代口径未定，却已经把具体官名写进标题和简介
- 核心冲突依赖官职压制，但官职真值表为空
- 同一角色同时背了互不相容的制度身份
- 权力推动靠“这官听起来很大”，而不是靠明确的上下级链

建议状态：

- `packaging-needs-update: yes`
- `submission-status: revise-then-submit` 或 `do-not-submit`

---

## 推荐资料层次

### 概览层

- `中国古代职官`
- `品秩`
- 各品级条目
- 具体官职条目

用途：

- 先确认术语边界
- 先看这一官名是不是存在于目标制度里
- 先看它属于中央、地方、军中、东宫、内廷还是命妇体系

### 确认层

- 官制史综述
- 具体朝代官职表
- 制度史研究
- 地方政治结构研究

用途：

- 锁“这个官在该朝到底归谁”
- 锁“品秩和实权有没有偏差”
- 锁“同知、知州、巡抚、总督这类地方层级如何串起来”

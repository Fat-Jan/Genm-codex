# 角色命名智能生成指南

## 当前 Fanqie-first 优先用法

如果项目当前目标是：

- 番茄优先
- 先做更容易出稿和验证的数据线

那么这份命名指南的默认优先级应调整为：

1. **当前内容桶**
2. **当前 tagpack**
3. **平台可读性**
4. **题材风味**

也就是说：

- 先问“这个名字像不像这个桶”
- 再问“这个名字美不美”

当前最优先参考桶：

- `宫斗宅斗`
- `现实情感`
- `现言甜宠`
- `传统玄幻`

---

## Fanqie-first 命名规则

### 一、共通规则

- 优先可读、可记、可分辨
- 不用生僻字
- 不追求“文艺感”压倒识别度
- 主角名最好一眼能和同场主要角色区分开
- 如果标题、简介、内容桶都偏强点击，名字不要再故作晦涩

### 二、按桶区分

#### 1. 宫斗宅斗

女主：

- 要稳、冷、能压场
- 可用：
  - 云、昭、宁、衡、清、绾、瑜、微、瑶、令

男主 / 权力角色：

- 要带权柄感、秩序感、距离感
- 可用：
  - 沉、珩、晏、霆、衍、玠、祁、珣、承、执

配角：

- 嫡庶线、继母线、庶妹线要一眼分辨
- 不要一堆“婉、柔、雪、月”堆在一起
- 如果要用族谱感、世家感命名，先确认这个家族是否真的设了行辈 / 谱字
- 不要默认所有闺阁女子都跟男性角色共用同一辈字

不推荐：

- 太甜
- 太软
- 太像校园 / 都市昵称

#### 2. 现实情感

女主：

- 要像现实里真的会出现的人名
- 但不能俗到没有记忆点
- 可用：
  - 知意、晚宁、亦宁、清禾、思妍、舒宁、嘉禾

男方 / 婆家线：

- 更偏现实常见名
- 这样压迫感更真实
- 可用：
  - 承礼、明哲、嘉伟、远航、立成、志远

不推荐：

- 过于古风
- 过于豪门偶像化
- 过于“网文感”

#### 3. 现言甜宠

女主：

- 要轻快、利落、带一点精致感
- 可用：
  - 林栀、温黎、许棠、姜宁、时晚、沈漾

男主：

- 要有都市高位感，但不宜太玄幻
- 可用：
  - 陆沉、周叙、裴砚、谢临、沈既白、贺行舟

不推荐：

- 过于土豪总裁感的“天、龙、霸”
- 过于古风或修仙化的字

#### 4. 传统玄幻

主角：

- 要有成长性、识别度、升级感
- 可用：
  - 沈照、陆玄、顾承渊、林渡、陈长川

配角：

- 宗门、长辈、同门、反派名字要层次分明

不推荐：

- 纯模板化“傲天流”堆字
- 所有人都像一个生成器里出来的名字

## Fanqie-first 命名输出要求

当技能给出命名建议时，至少应该说明：

1. 名字为什么适合当前桶
2. 名字在当前 tagpack 下传达什么风味
3. 名字是否过于常见、过于网文、过于文艺
4. 是否符合标题 / 简介 / 主角定位

### 投前命名风险门

如果这个名字会进入：

- 书名
- 简介
- 总纲
- 角色卡
- 投稿样本

那么不要只给“好不好听”的判断，还要补一层风险结论：

- `search_scope`
  - 仅项目内
  - 项目内 + 现有包装
  - 项目内 + 外部公开检索
- `name_risk_level`
  - low
  - medium
  - high
- `collision_notes`
  - 是否撞到同桶常见热门人名
  - 是否和现有书名 / 设定组合过近
  - 是否只是从热门字池机械拼接

最小检查顺序：

1. 先搜当前仓库是否已有相同主角名
2. 再看当前桶里是否已经有强绑定的同类名字组合
3. 如果用户目标是投稿或外部展示，再做一次外部公开检索

出现以下任一项时，默认不推荐直接定名：

- 主角全名已和相近题材公开文本高度绑定
- 标题关键词 + 主角名一起搜，已经能命中明显相近 premise
- 整套核心角色名都像从同一个高频字库机械拼出来

### 古代题材额外注意

- 若项目要写“排辈感”，先区分：
  - `行辈 / 字辈`
  - `排行称呼`
  - `闺名 / 谱名 / 字`
- 不要把三者混成一个系统
- 生成女性名字时，优先确认：
  - 她在正文里主要以名字出现，还是以称谓出现
  - 她是否真的需要与家族男性共享谱字
  - 她的名字是否会因过度追求“世家味”而与其他角色失去区分度

## 平台差异化命名策略

### 起点中文网（男频导向）

**男性角色命名风格**
- **核心特征**: 霸气、强势、有力量感
- **常用字库**: 轩、辰、寒、傲、天、云、风、龙、虎、战
- **命名模式**:
  - 单姓 + 单名：李轩、王辰、张寒
  - 单姓 + 双名：李傲天、王云龙、张战风
  - 复姓 + 单名：欧阳轩、司马辰、独孤寒
  - 复姓 + 双名：欧阳傲天、司马云龙

**女性角色命名风格**
- **核心特征**: 清冷、神秘、有仙气
- **常用字库**: 雪、月、花、梦、诗、雨、心、霜、冰、露
- **命名模式**:
  - 单姓 + 单名：李雪、王月、张花
  - 单姓 + 双名：李梦雪、王诗月、张雨心
  - 复姓 + 单名：欧阳雪、司马月

## 命名禁忌与注意事项

### 避免使用的字词
- **过于俗气**：财、富、贵、发、达
- **负面含义**：死、病、穷、丑、恶
- **敏感词汇**：政治、宗教相关词汇
- **生僻字**：影响读者阅读体验

### 平台特定禁忌

**起点中文网**
- 避免过于女性化的男性名字
- 避免过于普通的修仙者名字

### 番茄 / 投稿前额外禁忌

- 不要把“热门关系词标题”再叠加“高频古言女主名”，让整套外观过于像已有爆款
- 不要只因为某个名字在当前桶里常见，就默认它安全
- 不要在未做最小检索前，把主角全名写死进包装主方案
- 主角名字要有辨识度

**红袖读书**
- 避免过于霸气的女性名字
- 避免过于男性化的女性名字
- 名字要符合角色身份

**知乎**
- 避免过于夸张的名字
- 避免网络小说风格的名字
- 名字要符合现实逻辑

**豆瓣阅读**
- 避免过于俗套的名字
- 避免过于商业化的名字
- 名字要有文化内涵

## 智能生成算法

### 名字生成规则

1. **姓氏选择**
   - 常见姓氏：李、王、张、刘、陈、杨、赵、黄、周、吴
   - 复姓：欧阳、司马、上官、诸葛、东方、南宫
   - 地域姓氏：根据故事背景选择相应地域的姓氏

2. **名字组合**
   - 单字名：简洁有力，适合主角
   - 双字名：丰富多样，适合配角
   - 三字名：特殊身份，如复姓+双名

3. **音韵搭配**
   - 平仄搭配：避免全部平声或仄声
   - 韵母搭配：避免韵母相同
   - 声母搭配：避免声母相同

4. **意义关联**
   - 角色性格：名字含义与性格相符
   - 角色身份：名字风格与身份相符
   - 故事背景：名字符合时代背景

### 生成示例

**输入参数**：
- 平台：起点中文网
- 性别：男
- 身份：主角
- 题材：玄幻

**生成结果**：
- 李傲天、王云龙、张战风、刘轩辰、陈寒云

**输入参数**：
- 平台：红袖读书
- 性别：女
- 身份：女主角
- 题材：现代言情

**生成结果**：
- 李雨柔、王雪婉、张花静、刘月诗、陈梦心

## 使用建议

1. **主角命名**：要独特、有记忆点、符合角色定位
2. **配角命名**：要多样化、避免重复、符合身份
3. **反派命名**：要有特点、容易识别、符合邪恶形象
4. **群演命名**：要简单、普通、不影响阅读

通过这套命名系统，可以为不同平台、不同题材的角色生成符合平台特色和读者喜好的名字，提高作品的平台适配性和读者接受度。

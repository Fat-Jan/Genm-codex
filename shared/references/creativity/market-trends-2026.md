## 七、平台签约标准（最新）

### 番茄小说签约标准

```yaml
签约机会:
  - 2万字: 第1次申请机会
  - 5万字: 第2次申请机会
  - 8万字: 第3次申请机会

关键要求:
  - 3章内必有爽点（核心要求）
  - 8万字通过率比6000字高40%
  - 安全筛查 + 内容评估双审核

审核标准:
  黄金三章:
    - 前500字有冲突/危机
    - 主角前300字出场
    - 金手指3章内展现
  内容质量:
    - 爽点密度 1.2-1.5/千字
    - 无AI写作痕迹
    - 人设有记忆点
```

### 起点签约标准

```yaml
签约机制:
  - 字数要求: 3-5万字可申请
  - 以作品质量为主，不以字数为准
  - 黄金三章是关键

黄金三章要求:
  - 开篇200字内主角出现（最重要）
  - 3章内让主角解决一次矛盾
  - 不要有大量不着边际的内容
  - 人物不能太多（读者记不住）
  - 不能有太多水文
  - 留悬念钩子

审核重点:
  - 题材或设定需有创新点
  - 世界观完整且有深度
  - 人物智商在线
  - 每阶段至少看到3个不同要素（技能、装备、丹药等）
  - 避免剧情重复
```

### 七猫签约标准

```yaml
爽点要求:
  - 爽点密度: 1.3-1.8/千字（最高要求）
  - 每500字一个小爽点
  - 每章一个大爽点

开篇要求:
  - 第1章必须有冲突或危机
  - 金手指必须在3章内展现
  - 第3章前必须有第一次小打脸

人物要求:
  - 主角必须果断，不圣母不纠结
  - 冲突密集且直接
  - 打脸要狠要快

留存率基准:
  - 头部作品3万字留存率: 30%-40%
  - 新人作者曝光和读者基础较弱

更新要求:
  - 日更4000字以上
  - 稳定更新不断更

签约方式:
  - 全保底: 千字20-500元
  - 纯分成: 阶梯全勤
  - 超级分成: 千字10-30，最高可达千字1000
```

## 八、AI 写作检测与应对

### 平台AI检测现状

```yaml
检测平台:
  - 朱雀AI: 检测AI程度
  - 各平台内置检测: 番茄、七猫、起点、晋江均已上线

检测特征:
  - 句式过于规整对称
  - AI高频词: "淡淡的"、"微微"、"不禁"、"顿时"
  - 三段式结构
  - 逻辑过于严密
  - 缺乏口语化表达
```

### 去AI味道方法

```yaml
方法一: 人设立场法
  让AI扮演普通大学生口吻修改
  打破严密逻辑，增加随性表达

方法二: 句式重构法
  打破简单规整句式
  使用复杂嵌套结构
  长短句错落有致
  主被动语态自然切换

方法三: 追加优化法
  第一次: "去掉AI痕迹，用口语化重写"
  第二次: "还有AI味，继续优化，降到10%以下"
  第三次: 继续鞭策直到满意

方法四: 工具辅助
  检测: 朱雀AI检测AI程度
  消痕: 智语写作AI消痕功能、笔灵降AI
  验证: 再次检测确认

方法五: 核心指标调整（朱雀检测）
  - 句子长度标准差: 必须>1.5（AI文本<1.5）
  - 困惑度: 提高（AI困惑度低，过于可预测）
  - 突发性: 增加句子长度变化
  - 词汇灵活度: 减少重复，增加表达多样性
```

### AI痕迹高频词替换

| AI高频词 | 替换建议 |
|---------|---------|
| 淡淡的 | 删除或具体化（如"栀子花香"） |
| 微微 | 删除或具体描写 |
| 不禁 | 删除，直接描写动作 |
| 顿时 | 删除，直接描写结果 |
| 猛然 | 删除或换词 |
| 竟然 | 删除或换"居然" |
| 一丝 | 删除或具体化 |

## 九、可执行建议

### 低风险创新（推荐）

- 视角微调：第一人称 → 第二人称
- 机制创新：系统流 + 新规则
- 人设反转：传统配角当主角

### 中风险创新

- 叙事结构：双线并行、倒叙
- 多主角：群像剧
- 类型融合：悬疑 + 都市

### 高风险创新

- 全新题材：无对标作品
- 实验叙事：非线性、碎片化
- 反套路到底：颠覆所有预期

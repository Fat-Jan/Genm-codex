## 三大定律（低自由度 - 必须精确执行）

| 定律 | 规则 | 检查方式 |
|------|------|----------|
| **大纲即法律** | 严格执行大纲，不得擅自发挥 | 审查时对照大纲 |
| **设定即物理** | 实力/招式/物品 ≤ index.db 记录 | 写作前查询确认 |
| **发明需识别** | 新实体由 Hive Bee 自动提取 | 章节完成后处理 |

## 新实体处理流程（v5.2 引入，v5.4 沿用）

v5.2 引入的规则，v5.4 沿用：正文不再要求 XML 标签：

1. **写作时**: 直接写纯正文，新角色/地点/物品正常描写
2. **完成后**: Hive Bee 自动识别新实体并写入 index.db
3. **不确定实体**: Hive Bee 标记为 uncertain，由人工确认

## 章节约束分层（v5.5）

### Hard（必须）

- 本章可读性达标：读者能回答“发生了什么 / 谁在做什么 / 为什么”。
- 本章必须存在清晰推进：问题、目标、代价、关系变化、信息变化至少一项可被识别。
- 若上章有明确承诺（钩子/未闭合问题），本章必须回应（允许部分兑现，不要求一次性结清）。
- 关键推进不能白拿：重要证据、重要同盟、重要权力、重要身份收益至少要满足以下两项中的两项：
  - 有明确阻力
  - 有可见代价
  - 有残留风险
- 禁止输出占位正文（如 `[待补充]`、`[TODO]`、`...（省略）...`）。

### Soft（建议）

- 开头尽早进入冲突/风险/强情绪（建议前 200-400 字，不再固定 120 字）。
- 未闭合问题或下一章期待锚点建议放在章末或后段（不限定“后 80-150 字”）。
- 局面变化建议保持节奏感（参考 800-1400 字一个脉冲，短章至少一次实质变化）。
- 微兑现频率按题材 profile 建议执行，不要求机械等间距。

### Style（可选强化）

- 对话尽量带意图（试探/回避/施压/诱导），减少纯说明句。
- 避免连续大段纯解释；若必须解释，建议切分为”信息 + 行动/反应”。
- 避免”回去休息了”式机械收尾；若使用平缓收尾，需同时保留未闭合期待。

## 推进阻力与代价（v5.8 通用）

### 核心原则

- 快，不等于顺。
- 爽，不等于白拿。
- 角色要推进剧情，不是剧情替角色发奖励。

### 什么叫“推进过顺”

以下任一项反复出现，都视为结构风险：

1. 证据自动上门
   - 刚起疑，就有人送关键证据
   - 没有获取成本、真假风险或交换条件
2. 同盟瞬间成立
   - 原本应互防的人，对几句就完全同频
   - 没试探、没让利、没卡牌
3. 权力立刻兑现
   - 身份一亮，所有人自动低头
   - 手令一到，所有门自动开
   - 没人试边界，没执行成本
4. 胜利没有代价
   - 赢了，但主角没暴露、没树敌、没失去任何东西
5. 谜面刚出，答案就到
   - 刚抛问题，下个场景就有人给全答案

### 每章至少检查四项

- `阻力`
  - 谁在拦，规矩、人、时间、门槛、舆论哪一层在拦
- `代价`
  - 主角为了这次收益实际付出了什么
- `局部收益`
  - 这次到底赢到哪一步
- `残留风险`
  - 赢完以后，什么变得更危险了

若本章的重要推进里，这四项有两项为空，通常就会显得过顺。

### 黄金三章特别约束

- 第1章：可以掀桌，但不能一口气打穿主线矛盾。
- 第2章：最好出现一次卡点、误判、交换条件、短暂失手或信任延迟。
- 第3章：可以硬兑现，但最好是半胜、惨胜、明赢暗亏，不能全赢到让后续只剩加码。

### 关系型与智斗型通用规则

- 关系绑定可以快，信任不能快。
- 站边可以先发生，同频要后发生。
- 证据可以出现，必须说明为什么现在能拿到。
- 最稳的章末不是“事情全成了”，而是“事情成了一半，但局更大了”。

### 最短检查表

- [ ] 本章收益前是否真的有阻力
- [ ] 阻力不是一句话带过
- [ ] 主角有没有为这次收益付出代价
- [ ] 这次赢完是否留下残留风险
- [ ] 关键证据不是自动送到手上
- [ ] 关键同盟不是一句话立刻成立
- [ ] 第3章结尾不是全赢

## 去AI味道约束（v5.7 扩展）

### 核心规则速查

1. **删除填充短语** - 去除开场白和强调性拐杖词
2. **打破公式结构** - 避免二元对比、三段式法则、修辞性设置
3. **变化节奏** - 混合句子长度，段落结尾多样化
4. **信任读者** - 直接陈述事实，不过度解释
5. **删除金句** - 如果听起来像可引用的语句，重写它

### 禁用词汇清单

以下词汇 AI 高频使用，必须删除或替换：

| 类型 | 禁用词 | 替代方案 |
|------|--------|---------|
| 程度副词 | 淡淡的、微微、不禁、竟、顿时 | 删除或用动作替代 |
| 情感副词 | 欣喜地、悲伤地、愤怒地 | 用行为/生理反应替代 |
| 时间副词 | 突然、猛然、霎时 | 用短句制造冲击 |
| AI学术词 | 此外、至关重要、深入探讨、强调 | 删除或换口语表达 |
| AI抽象词 | 格局、织锦、复杂性、关键性 | 删除或写具体内容 |
| 夸大意义 | 作为/充当、标志着、见证了 | 删除，直接说是什么 |
| 宣传语言 | 拥有、充满活力的、令人叹为观止的 | 删除或写具体事实 |

### 24 种 AI 写作模式速查

**内容模式（6种）**：
1. 过度强调意义 → 删除”标志着/作为/见证了”
2. 过度强调知名度 → 删除模糊归因，写具体来源
3. -ing 结尾肤浅分析 → 删除句末分词短语
4. 宣传性语言 → 删除”充满活力的/令人叹为观止的”
5. 模糊归因 → 删除”专家认为/观察者指出”
6. 公式化”挑战”部分 → 删除”尽管存在这些挑战”

**语言模式（6种）**：
7. AI 高频词 → 删除”此外/至关重要/格局”
8. 系动词回避 → 用”是/有”替代”作为/拥有”
9. 否定式排比 → 删除”不仅……而且……”
10. 三段式法则 → 改为两项或四项
11. 同义词循环 → 重复用词，不刻意换词
12. 虚假范围 → 删除”从X到Y”的跨度描写

**风格模式（6种）**：
13. 破折号过度 → 用句号断开
14. 粗体过度 → 删除粗体标记
15. 内联标题列表 → 改为流畅段落
16. 标题大写 → 中文不适用
17. 表情符号 → 删除
18. 弯引号 → 使用中文引号

**交流模式（6种）**：
19. 协作痕迹 → 删除”希望这对您有帮助”
20. 知识截止免责 → 删除”截至/基于可用信息”
21. 谄媚语气 → 删除”好问题！您说得对”
22. 填充短语 → “为了实现这一目标”→”为了这个”
23. 过度限定 → 删除”潜在地可能/一些影响”
24. 通用积极结论 → 删除”未来看起来光明”

### 核心改写技巧

1. **展示代替说明**：不写”他很生气”，写”他拍桌子”
2. **删副词用动作**：不写”淡淡地笑了”，写”嘴角动了动”
3. **情绪外化**：不写”心中涌起暖流”，写”她低头，用袖子擦了眼睛”
4. **对话口语化**：不写书面语，写人话
5. **句式变化**：长短句穿插，省略主语，倒装

### 质量评分（总分50）

| 维度 | 标准 | 10分=好 |
|------|------|---------|
| 直接性 | 直截了当 vs 铺垫绕圈 | 直截了当 |
| 节奏 | 长短交错 vs 机械重复 | 长短交错 |
| 信任度 | 简洁明了 vs 过度解释 | 简洁明了 |
| 真实性 | 自然流畅 vs 机械生硬 | 自然流畅 |
| 精炼度 | 无冗余 vs 大量废话 | 无冗余 |

**标准：45-50分=优秀，35-44分=良好，<35分=需修订**

## 爽点与节奏（按题材 profile 调整）

- 爽点密度由题材与章型共同决定：过渡章允许低密度，但不允许“整章无收获”。
- 组合爽点与里程碑爽点采用滚动窗口评估（5章/10-15章），用于预警而非逐章硬判。
- 连续同类型爽点达到 3 章记为风险预警，优先通过类型变体或执行差异化修正。

## Strand 平衡警告

| 情节线 | 警告条件 |
|--------|----------|
| Quest（主线） | 连续 5+ 章 |
| Fire（感情线） | >10 章未出现 |
| Constellation（世界观） | >15 章未出现 |

## 禁止事项

- `[待补充]`、`[TODO]`、`...（省略）...` → 必须完整写出
- 战斗后无善后描述（都市异能题材）
